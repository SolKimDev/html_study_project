  1. 챕터 3-1. 이미지 ▽

    • ⟨img⟩ 태그는 웹 문서에 이미지를 삽입할 때 사용하며, GIF, JPG/JPGE, PNG 파일을 이미지로 이용할 수 있습니다. 기본형은 아래와 같습니다.

      ⟨ img src="경로" [속성="값"] ⟩
      • alt 속성은 낭독기 및 이미지 로드 에러 시 표시되는 대체 텍스트입니다.
      • figure와 figcaption은 ⟨p⟩대신 이미지에 직접 사용할 수 있는 캡션 옵션입니다.
        우선 figure로 대상을 지정한 뒤, figure 태그 안에 figcaption 태그를 사용해 설명 글을 표시합니다.
        ex ) : ⟨figure⟩ 요소 ⟨figcaption⟩ 설명 글 ⟨/figcaption⟩ ⟨/figure⟩

  2. 챕터 3-2. 링크 ▽

    ⟨a⟩태그 안에서 사용할 수 있는 주요 속성은 다음과 같습니다.

    • href : 링크한 문서나 사이트의 주소를 입력합니다.
    • target : 링크한 내용이 표시될 위치(현재 창 또는 새 창)를 지정합니다.
      target 속성의 값은 다음과 같습니다.
      • _blank : 링크 내용이 새 창이나 새 탭에서 열립니다.
      • _self : target 속성의 기본값으로, 링크가 있는 화면에서 열립니다.
      • _parent : 프레임을 사용했을 때 링크 내용을 부모 프레임에 표시합니다.
      • _top : 프레임을 사용했을 때 프레임에서 벗어나 링크 내용을 전체 화면에 표시합니다.
    • download : 링크한 내용을 보여 주는 것이 아니라 다운로드합니다.
    • rel : 현재 문서와 링크한 문서의 관계를 알려줍니다.
    • hreflang : 링크한 문서의 언어를 지정합니다.
    • type : 링크한 문서의 파일 유형을 알려줍니다.

    ⟨a⟩태그는 또한, 페이지 내에서 이동할 때에도 사용할 수 있습니다. 기본형은 다음과 같습니다.

    ex ) : ⟨태그 id="앵커 이름"⟩ 텍스트 또는 이미지 ⟨/태그⟩ or
    ⟨a href="#앵커 이름"⟩ 텍스트 또는 이미지 ⟨/a⟩ a태그의 href 속성에 #id만 입력해주면 쉽게 사용할 수 있지만 단점이 있다. 첫번째는 해당영역으로 바로 이동되어 부드러운 스크롤 효과가 없다는 점이다. 두번째는 url 주소에 #id명의 해쉬가 생성되는데 자바스크립트 history.back()이나 history.go(-1)를 이용하여 뒤로가기를 구현했다면 해쉬로 인해 2번 클릭해야 한다는 문제가 발생한다. 하지만 해쉬를 이용해 history를 조작할 수 있다는 점에서도 해쉬는 장점이 될 수 있기 때문에 적절한 순간에 사용하면 매우 유용한 기능이다. 고급 스크롤 이용하려면 data-* 이용하기. ->https://phrygia.github.io/2020-12-02/scroll

    현재 상위 메뉴 선택에 적용되어 있으며, a href='#id'의 형태로 적용되어 있습니다.


    이미지 맵이란, 하나의 이미지를 구역에 따라 나누어, 링크를 다르게 적용하는 것입니다.
    이미지에 이미지맵을 적용하기 위해서는 ⟨⟩태그와 ⟨⟩태그를 이용하여 미리 맵을 만들어 둔 뒤, usemap 속성을 통해 이미지에 map을 적용시키는 방법을 이용합니다.
    기본형은 다음과 같습니다.

    ▶이미지 맵 기본형 (펼치기)

    ⟨map name="맵이름"⟩
      ⟨area⟩
      ⟨area⟩
      ......
    ⟨/map⟩

    ⟨img src="이미지 파일" usemap="#맵이름"⟩

    이미지 맵으로 사용할 이미지에 영역을 표시할 때에는 ⟨area⟩태그를 사용하는데, ⟨area⟩ 태그에서 사용할 수 있는 속성은 다음과 같습니다.

    • alt : 대체 텍스트를 지정합니다.
    • coords : 링크로 사용할 영역을 시작 좌표와 끝 좌표를 이용해 지정합니다.
    • download : 링크를 클릭했을 때 링크 문서를 다운로드합니다.
    • href : 링크 문서(사이트)경로를 지정합니다.
    • media : 링크 문서(사이트)를 어떤 미디어에 최적화시킬지 지정합니다.
    • rel : 현재 문서와 링크 문서 사이의 관계를 지정합니다.
    • shape : 링크로 사용할 영역의 형태를 지정합니다.
    • target : 링크를 표시할 대상을 지정합니다.
    • type : 링크 문서의 미디어 유형을 지정합니다.

    이미지 맵의 적용 예는 다음과 같습니다. => 페이지 열기 (팝업)
    '최신간' 부분과 '분수의 덧셈' 부분에 링크 적용되어 있습니다.

    챕터 3-3. SVG 이미지 ▽

    SVG 이미지란 Scalable Vector Graphics의 약자로, 웹 상에서 벡터 이미지를 표현하기 위에 이용됩니다.
    SVG 이미지는 2001년 웹 표준으로 지정된 방식으로서, 마크업 형식으로 이미지의 벡터를 서술하기 때문에, 웹 상에서 벡터 이미지를 표현하기 위해 ⟨img⟩태그를 이용해 *.svg 파일 형태로 직접 삽입할 수도 있지만, 태그를 이용해 직접 만들 수도 있습니다.

    다만, IE8 이하에서는 SVG를 지원하지 않으므로, 지원 여부에 따라 대체 로딩 로직을 구현하는 등의 방법을 마련해야 할 것입니다.

    이에 대해 자세한 내용은, 교재 123-124p에 기술되어 있습니다. (Modernizr.js 이용)
