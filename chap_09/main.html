  1. CSS 포지셔닝과 주요 속성들

    CSS 포지셔닝이란 브라우저 화면 안에 각 콘텐츠 영역을 어떻게 배치할지를 결정하는 것으로, float속성과 position 속성 등을 이용하며, 여기에 박스 모델의 padding이나 border 속성 등을 포함해 전체적인 레이아웃을 결정합니다.
    이번 장에서는 CSS의 포지셔닝을 이용해 웹 문서의 레이아웃을 구성하는 방법을 알아봅니다.

    • box-sizing 속성 : 박스 너비 기준 정하기

      CSS의 width 속성의 경우 콘텐츠 영역의 너비를 나타내기 때문에 해당 요소에 적용한 패딩이나 테두리 크기는 따로 계산해서 배치해야 합니다.
      이럴 때 box-sizing 속성을 사용하면 콘텐츠 영역의 너비에 패딩과 크기까지 합쳐서 width 속성을 지정할 수 있습니다.
      기본형은 다음과 같습니다.

      box-sizing: content-box | border-box

      박스 모델의 마진은 width 속성에 포함되지 않습니다.

      각 속성에 대한 설명은 다음과 같습니다.

      • content-box : 기본값으로, width 속성 값을 콘텐츠 영역 너비 값으로 사용합니다.
      • border-box : width 속성 값을 콘텐츠 영역에 테두리까지 포함한 박스 모델 전체 너비 값으로 사용합니다.
        따라서, 박스 모델의 너비 값(width) = 콘텐츠 영역 + 좌우 패딩 + 좌우 테두리가 됩니다.

    • float 속성 : 왼쪽이나 오른쪽으로 배치하기

      float 속성은 주로 나란히 놓을 수 없는 블록 요소를 왼쪽 구석이나 오른쪽 구석에 나란히 배치할 때 사용하는데, float 속성을 사용하면 너비 값은 콘텐츠를 표시할 때 필요한 만큼만 차지하고, 다른 요소가 들어올 만큼의 공간을 비워 둡니다.
      또한, float 속성을 이용하면 그 다음에 넣는 요소들에도 똑같은 속성이 전달되며, float속성이 더이상 필요하지 않을 경우 이를 해제해 주어야만 합니다.
      float 속성을 사용함에 있어서 유의할 점은, float 속성은 요소의 위에 떠 있는 속성이기 때문에, float를 적용한 부모 속성의 너비가 기본적으로 0으로 인식됩니다.

      float 속성을 사용하면 그 주변을 다른 요소가 감쌉니다. 따라서 이미지와 텍스트를 나란히 표시하려 할 때는 이미지에 float 속성을 사용합니다.
      이미지를 배치한 후에는 적당한 간격을 유지하기 위해 margin 속성도 함께 사용하는 것이 권장됩니다.

      float에서 사용 가능한 속성은 left, right, none 속성으로, 기본형은 다음과 같습니다.

      float: left | right | none

    • clear 속성 : float 속성 해제하기

      float 속성이 더 이상 유효하지 않을 경우, 이 속성 다음부터의 float 속성은 해제됩니다.
      float: left를 이용해 왼쪽으로 배치했다면 float:left로 종료하고, float: right를 이용했다면 clear:right를 이용해 무효화시킵니다.
      float 속성 값이 left인지 right인지 상관없이 무조건 기본 사태로 되돌리고 싶다면 clear: both를 이용하면 됩니다.
      기본형은 다음과 같습니다.

      clear: none | left | right | both

      ※clear 심화※

      float를 이용 후, 이를 무효화하지 않고 div 등의 요소를 놓을 경우, float가 영역을 차지하지 않는다고 여기고 해당 영역과 겹치기 때문에, 이를 해소할 방법이 필요합니다.
      이 문제를 해결하기 위한 방법들이 어떠한 것들이 있는지 다양한 방법들을 알아봅시다.

      1. float로 대응하기

        앞의 요소가 float인 상태에서 뒤따라오는 요소가 float가 아닐 경우 이런 문제가 발생하기 때문에, 그 다음 요소 역시 float 상태로 바꾸어 주면 해결됩니다.
        다만 이는 그 뒤의 요소들에게도 동일한 문제가 발생하기 때문에 완전한 해결책이라고 볼 수 없습니다.

      2. 부모에게 overflow:hidden 사용하기

        자식요소가 float인 부모에게 overflow:hidden을 주면 부모가 자식을 담아냅니다. 다만, 이 방법은 내용이 넘치면 안 보이기 때문에 한계가 있습니다.
        예시는 다음과 같습니다.

        ⟨style⟩
        div.wrap{overflow: hidden}
        div.box1, div.box2 {float: left}
        ⟨style⟩
        ⟨div class="wrap"⟩
        ⟨div class="box1"⟩⟨/div⟩
        ⟨div class="box1"⟩⟨/div⟩
        ⟨/div⟩

      3. 빈 요소에 clear:both 넣기

        float된 요소 밑에 빈 임의의 요소를 만들어 clear:both를 줍니다. clear는 취소하다는 뜻으로 float를 취소시킵니다.
        이건 마크업의 흐름을 깨기 때문에 권장하는 방법은 아닙니다.
        예시는 다음과 같습니다.

        ⟨style⟩
        div.box1, div.box2{float: left}
        br.clear{clear:both}
        ⟨style⟩
        ⟨div class="box1"⟩⟨/div⟩
        ⟨div class="box1"⟩⟨/div⟩
        ⟨br class="clear"⟩

      4. 가상요소 ::after 사용하기

        부모요소에게 ::after를 사용하며 가상요소를 만들어 줍니다. 그리고 가상요소에게 display:block 요소를 적용해 한 줄 가득차게 합니다.
        그리고 빈 콘텐츠를 넣어주고, clear를 적용시켜줍니다. 부모 뒤에 보이지 않는 가상요소를 만들어 뒤에 요소가 위로 올라가지 않게 해줍니다.
        가장 많이 사용하는 방법으로, 예시는 다음과 같습니다.

        ⟨style⟩
        div.clearfixed::after{display:block;content:""; clear:both}
        div.box1, div.box2{float: left}
        ⟨style⟩
        ⟨div class="clearfixed"⟩
        ⟨div class="box1"⟩⟨/div⟩
        ⟨div class="box1"⟩⟨/div⟩
        ⟨/div⟩

    • position 속성 : 배치 방법 지정하기

      position 속성은 웹 문서 안의 요소들을 자유자재로 배치해 주는 속성으로 HTML과 CSS를 이용해 웹 문서를 만들 때 중요하게 사용하는 속성 중 하나입니다.
      position 속성을 이용하면 텍스트나 이미지를 나란히 배치할 수 있고 여러 개의 요소를 가로나 세로로 원하는 위치에 배치할 수도 있습니다. 이렇게 요소를 다양하게 배치하려면 position 속성에서 사용하는 속성 값의 특성을 이해하고 필요에 따라 적절한 속성 값을 선택해야 합니다.
      position 속성의 기본형은 다음과 같습니다.

      position: static | relative | absolute | fixed

      각 속성에 대한 설명은 다음과 같습니다.

      • static : 기본값으로, 요소를 문서의 흐름에 맞추어 배치합니다.
      • relative : 이전 요소에 자연스럽게 여결해 배치하되, 위치를 지정할 수 있습니다.
      • absolute : 원하는 위치를 지정해 배치합니다.
      • fixed : 지정한 위치에 고정해 배치합니다. 화면에서 요소가 잘릴 수도 있습니다.

      position 속성 중 static을 제외한 나머지 속성 값에서는 좌표를 이용해 각 요소의 위치를 조절할 수 있습니다.
      이 때, 위치는 top과 bottom, left, right로 지정합니다.
      top이란 위쪽에서 얼마나 떨어져 있는지, bottom은 아래에서 얼마나 떨어져 있는지를 나타냅니다.
      좌푯값은 양수와 음수 모두 사용할 수 있습니다.
      각 속성 값에 대한 자세한 설명은 다음과 같습니다.

      • static 속성 값 : 문서의 흐름대로 배치하기
        box1
        box2

        static은 position 속성의 기본값으로 요소를 나열한 순서대로 배치하며 top이나 right, bottom, left같은 속성을 사용할 수 없습니다.
        단지 float 속성을 이용해 좌우로 배치할 수 있을 뿐입니다.

      • relative 속성 값 : 문서 흐름 따라 위치 지정하기
        box1
        box2

        relative 속성 역시 static에서 처럼 요소를 나열한 순서대로 배치합니다.
        그러나 이 속성이 static과 다른 점은 top, bottom, left, right와 같은 속성을 이용해 원래 배치되어야 할 위치에서 자리를 조금씩 옮겨서 배치할 수 있다는 점입니다.

      • absolute 속성 값 : 원하는 위치에 배치하기
        box1

        absolute 속성을 이용하면 문서의 흐름과 상관없이 left와 right, top, bottom 속성 값을 이용해 요소를 원하는 위치에 배치할 수 있습니다.
        이 때 기준이 되는 위치는 가장 가까운 부모 요소나 조상 요소 중 position 속성이 relative인 요소입니다.
        따라서, absolute 속성을 이용하기 위해서 부모 요소는 반드시 position: relative로 지정되어 있어야 합니다.

      • fixed 속성 값 : 브라우저 창 기준으로 배치하기
        box1
        I'm fixed!

        fixed 속성 값도 absolute 속성 값처럼 문서의 흐름과 상관없이 위치를 좌표로 결정하지만, 부모 요소가 아닌 브라우저 창이 기준이 됩니다.
        브라우저 창의 왼쪽 위 꼭지점을 원점으로 두고 좌표가 계산되며 한 번 배치되면 브라우저 창을 스크롤하더라도 계속 고정되어 표시됩니다.
        현재 오른쪽의 네비게이션 역시 fixed 속성이 적용되어 있습니다.

    • visibility 속성 : 요소를 보이게 하거나 보이지 않게 하기

      특정 요소를 화면에 보이게 하거나 보이지 않게 또는 겹치게 설정하는 속성입니다.
      그러나, display: none;과는 다르게 눈에 보이지는 않지만 실제로는 공간을 차지합니다.
      기본형은 다음과 같습니다.

      visibility: visible | hidden | collapse

      각 속성에 대한 설명은 다음과 같습니다.

      • visible : 화면에 요소를 표시합니다. 기본값입니다.
      • hidden : 화면에서 요소를 감추지만 크기는 유지할 수 있도록 하여 배치에 영향을 줍니다.
      • collapse : 표의 열, 행 그룹, 열 그룹 등에서 지정하면 서로 겹치도록 조절합니다. 그 외의 영역에서 사용할 시 'hidden'과 같습니다.
        표에서 collapse를 사용해 visibility를 조절하는 예시는 준비중에서 확인할 수 있습니다.

    • z-index 속성 : 겹친 요소 순서 정하기
      1
      2
      3

      요소 위에 요소를 겹쳐 쌓을 때, 쌓는 순서를 지정합니다.
      요소를 겹쳐 쌓기 위해서는 position 이 absolute 또는 relative 이고, z-index 가 auto 가 아닌 요소. 또는 position 이 fixed 또는 sticky 인 요소를 겹칠 수 있습니다. z-index 값이 작을수록 아래에 쌓이고, 클수록 위에 쌓입니다.
      z-index 값을 명시하지 않을 경우 맨 먼저 삽임된 요소가 1값을 가지며, 그 이후 값이 점점 커집니다.
      기본형은 다음과 같습니다.

      z-index: ⟨숫자⟩

      z-index 더 알아보기

      1. z-index는 설정한 z-index order 외에도, 웹 페이지의 natural stacking order에 영향을 받습니다.
      2. 요소의 위치를 설정하기 위해서는 static이 아닌 position에 relative,absolute와 같은 값을 설정해야 합니다.
      3. opacity 또는 transform같은 css 속성들을 설정하면 element가 새로운 stacking context에 배치됩니다.

  2. 다단으로 편집하기

    column 관련 속성으로 다단을 편집할 수 있습니다. 다만, 다단 관련 속성은 브라우저별 접두사를 붙여 사용해야 합니다.
    IE 11 이상과 Edge에서는 다단 속성을 지원하므로 접두사가 따로 필요하지는 않습니다.

    • column-width : 단의 너비 고정하고 다단 구성하기

      단을 나눌 때, 단의 너비를 고정해 놓고 화면을 분할하거나, 단의 개수를 고정해 놓고 화면을 분할하는 방법이 있습니다.
      단의 너비를 고정해 놓고 구성하는 방법은 화면이 커지면 단의 개수가 많아지고, 화면이 좁아지면 단의 개수가 줄어듭니다.
      다단으로 편집할 때 단의 너비를 고정해 놓으려면 column-width 속성을 이용합니다.

      기본형은 다음과 같습니다.

      column-width: ⟨단 크기⟩ | auto

      크기 속성을 이용하면 단 너비를 직접 지정할 수 있고, auto 속성을 이용하면 단의 개수(column-count)와 같은 다른 속성에 의해 단의 너비가 자동으로 계산됩니다.

    • column-count 속성 : 단의 개수 고정하고 다단 구성하기

      다단을 구성할 때 단의 개수를 고정하면 브라우저 창의 너비와 상관없이 단의 개수를 항상 일정하게 유지합니다.
      따라서 창의 너비에 따라 단의 너비가 자동으로 조절됩니다.

      기본형은 다음과 같습니다.

      column-count: ⟨단 갯수⟩ | auto

      auto 속성을 이용하면 단의 너비(column-width)와 같은 다른 속성에 의해 단의 너비가 자동으로 계산됩니다.

    • column-group 속성 : 단과 단 사이 여백 지정하기

      단과 단 사이의 여백을 지정합니다. 단과 단 사이에 구분선을 넣는다면 구분선도 이 여백 안에 들어갑니다.

      기본형은 다음과 같습니다.

      column-gap: ⟨여백 크기⟩ | normal

      normal로 설정하면 여백을 자동으로 지정합니다. W3C 권장 여백은 1em입니다.

    • column-rule 속성 : 구분선의 색상, 스타일, 너비 지정하기

      단과 단 사이에 구분선을 넣습니다.
      구분선의 속성을 한번에 지정하고 싶다면 column-rule 속성을, 개별 속성을 지정하고 싶다면 column-rule-( color | style | width ) 속성을 각각 사용합니다.

      기본형은 다음과 같습니다.

      column-rule: ⟨너비⟩ ⟨스타일⟩ ⟨색상⟩ or
      column-rule-style: none | hidden | dotted | dashed | solid | double | groove | ridge | inset | outset or
      column-rule-width: ⟨크기⟩ | thin | medium | thick or
      column-rule-color: ⟨색상⟩

    • break-after 속성 : 다단 위치 지정하기

      다단의 위치를 실제 콘텐츠 영역 내에 제한하는 등, 다단의 위치를 설정허기 위해 다단의 시작 위치를 지정합니다.

      기본형은 다음과 같으며, 속성에 따른 특성을 정리하면 다음과 같습니다.

      break-after : column | avoid-column
      break-before : column | avoid-column
      break-inside : column | avoid-column
      속성단 나눌 위치동작
      단 나눔단 나누지 않음
      break-before특정 요소 앞columnavoid-column
      break-after특정 요소 뒤
      break-inside특정 요소 안

    • column-span 속성 : 여러 단을 하나로 합치기

      다단을 구성하지 않고 단을 합칠 영역을 지정합니다. 기본형은 다음과 같습니다.

      column-span: 1 | all

      각 속성에 대한 설명은 다음과 같습니다.

      • 1 : 단을 하나만 합치는 것이므로 단을 합치지 않는 것과 같습니다. 기본값입니다.
      • all : 전체 단을 하나로 합쳐 표현합니다. 단의 일부만 합칠 수는 없습니다.

      • 다단 관련 속성을 모두 적용하면, 다음과 같은 모습을 확인할 수 있습니다.

      See the Pen by 솔🌲 (@wierd_sol) on CodePen.

  3. 표 스타일

    표 스타일을 활용하면 표의 크기, 테두리, 셀 테두리, 여백 등의 스타일을 다양하게 적용할 수 있습니다.

    • caption-side 속성 : 표 제목 위치 정하기

      표 제목은 caption 태그를 이용해 캡션으로 표시합니다.
      기본적으로 캡션은 표의 위쪽에 표시되지만, caption-side:bottom; 을 이용하여 캡션의 위치를 표 아래쪽으로 옮길 수 있습니다.
      기본값은 top이며, 기본형은 다음과 같습니다.

      caption-side: top | bottom

    • border 속성 : 표 테두리 스타일 결정하기

      표를 삽입할 때 기본적으로 table 태그의 border 속성을 이용해 ⟨table border="1"⟩처럼 사용하면 표에 테두리를 그릴 수 있습니다.
      여기에 CSS의 border 속성을 이용해 테두리의 색상이나 형태, 너비 등을 지정할 수 있습니다.
      table의 border와 CSS의 border 속성의 차이점은, table속성에서 border를 설정했다면 표의 바깥 테두리뿐만 아니라 셀의 테두리까지 모두 표시되지만,
      CSS를 이용해 테두리를 표시할 때는, 표의 바깥 테두리와 셀의 테두리는 별도로 지정해야 합니다.

    • border-collapse 속성 : 테두리 통합, 분리하기

      ⟨table⟩과 ⟨td⟩의 보더를 함께 설정하는 것과 같이, 표의 보더가 한 영역에 두 줄로 표시되는 경우가 있습니다.
      이 경우, border-collapse속성을 적용하면 인접한 두 줄의 보더를 겹칠 것인지, 따로 둘 것인지 설정할 수 있습니다. 기본값은 separate로, 기본형은 다음과 같습니다.

      border-collapse: collapse | separate

    • border-spacing 속성 : 인접한 셀 테두리 사이 거리 지정하기

      border-spacing 속성은 border-collapse:separate를 사용해 셀들을 분리했을 경우, 인접한 셀 테두리 사이의 거리를 지정합니다.
      border-spacing 속성 값은 1~2개를 지정할 수 있는데, 값이 하나라면 수평 거리와 수직 거리를 한꺼번에 지정한 것이고, 값이 2개라면 첫 번째 값은 수평 거리 값이고 두 번째 값은 수직 거리 값입니다.
      기본형은 다음과 같습니다.

      border-spacing: ⟨크기⟩

    • empty-cells 속성 : 빈 셀의 표시 여부 지정하기

      border-collapse:separate를 사용해 셀들을 분리했을 경우, empty-cells 속성을 사용해 내용이 없는 빈 셀들의 표시 여부를 지정합니다. 기본형은 다음과 같습니다.

      empty-cells: show | hide

      show 속성 hide 속성

    • width, height 속성 : 표 너비와 높이 지정하기

      표의 너비와 높이를 특별히 지정하지 않는다면, 셀의 크기는 내부의 콘텐츠의 크기만큼만 지정됩니다.
      표의 너비를 지정하고 셀의 너비를 지정하지 않는다면, 내용이 없을 경우의 표의 너비를 셀의 개수로 나누어 셀의 너비를 일정하게 지정하고, 내용이 있다면 내용에 따라 동일한 비율로 지정합니다.
      width값만 지정할 경우, 셀의 테두리와 셀의 내용 사이가 브라우저에서 지정한 값만큼만 떨어지기 때문에, 필요하다면 padding 속성을 이용해 여백을 조금 더 넣어 보기 좋게 꾸밀 수 있습니다.

    • table-layout 속성 : 콘텐츠에 맞게 셀 너비 지정하기
      한글로띄어쓰기없이길게붙여쓰기 long_description_without_space

      두 테이블은 모두 width: 200px이 적용되어 있으나, 영문 내용을 여백 없이 길게 입력할 경우, ⟨td⟩의 속성은 무시되고 영문 내용은 모두 한 줄로 표시됩니다.
      그리고 표의 전체 너비에 맞추기 위해 옆에 있는 셀의 너비가 그만큼 줄어들게 됩니다.
      이것은 처음 의도했던 표의 형태가 아니기 때문에 여백 없이 긴 영문 글자가 입력되더라도 CSS에서 지정한 width 값을 지켜야 합니다.

      한글로띄어쓰기없이길게붙여쓰기 long_description_without_space

      이 때, table-layout:fixed 속성을 이용해, 다른 셀의 너비 때문에 셀이 한쪽으로 몰리는 것을 방지할 수 있습니다.
      그러나, table-layout:fixed를 적용하면 셀 너비보다 긴 내용은 셀 밖으로 밀려나가 버립니다.
      각 셀의 너비를 고정한 상태에서 셀 너비 안에 셀 내용을 표시하려면 word-break:break-all 속성을 추가해야 합니다.
      또한, 예상하지 못했던 셀의 줄 바꿈이 발생할 수 있기 떄문에, 셀의 height 속성도 auto로 설정해야 합니다.

    • text-align 속성 : 셀 안에서 수평 정렬하기

      text-align 속성은 셀 안에서 텍스트의 수평 정렬 방법을 지정합니다.
      사용할 수 있는 값은 일반 텍스트 정렬에서와 같으며, 기본형은 다음과 같습니다.

      text-align: left | right | center

    • vertical-align 속성 : 셀 안에서 수직 정렬하기

      vertical-align 속성은 수직 정렬 방법을 지정하는 속성이기 때문에 inline이나 inline-block으로 배치한 요소의 세로 정렬 방법으로 지정합니다.
      기본형은 다음과 같습니다.

      vertical-align: baseline | top | bottom | middle | sub | super | text-top | text-bottom | ⟨길이 값⟩ | ⟨백분율 값⟩

      각 속성에 대한 설명은 다음과 같습니다.

      • baseline : 인라인 요소의 기준선을 부모 요소의 기준선(baseline)에 맞춥니다.
      • sub : 인라인 요소의 기준선을 부모 요소의 아래 첨자 위치에 맞춥니다.
      • super : 인라인 요소의 기준선을 부모 요소의 위 첨자 위치에 맞춥니다.
      • top : 인라인 요소의 윗부분을 부모 요소의 윗부분에 맞춥니다.
      • middle : 인라인 요소의 중앙 부분을 부모 요소의 기준선에서 x-height[?]의 반만큼 올려서 맞춥니다.
      • bottom : 인라인 요소의 아랫부분을 부모 요소의 아랫부분에 맞춥니다.
      • text-top : 인라인 요소의 윗부분을 부모 요소 글꼴의 윗부분에 맞춥니다.
      • text-bottom : 인라인 요소의 아랫부분을 부모 요소 글꼴의 아랫부분에 맞춥니다.
      • ⟨길이 값⟩ : 기준선을 0px로 생각하고, 길이 값이 양수면 기준선 위로, 음수면 기준선 아래로 지정한 크기만큼 옮깁니다.
      • ⟨백분율 값⟩ : 기준선을 0%로 생각하고, line-height의 몇%인지에 따라 양수면 위로, 음수면 아래로 옮깁니다.

      해당 속성을 표의 셀에서 사용할 경우, 기준선이나 위, 아래, 가운데 등으로 정렬할 수 있습니다. 상세한 속성은 다음과 같습니다.

      • baseline : 기본값으로, 셀의 기준선에 내용의 기준선을 맞춥니다.
      • top : 패딩의 위쪽 가장자리에 내용의 윗부분을 맞춥니다.
      • middle : 패딩 박스의 중앙에 내용을 맞춥니다.
      • bottom : 패딩의 아래쪽 가장자리에 내용의 아랫부분을 맞춥니다.
