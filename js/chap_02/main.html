  1. 객체

    JS도 자바와 마찬가지로 객체 기반 언어로, 객체는 기능 또는 속성을 가지고 있습니다.
    JS Object에서의 기능을 Method, 속성을 Property라고 합니다.

    • 객체의 종류

      자바스크립트의 객체는 크게 내장 객체, BOM, DOM으로 나눌 수 있습니다.

      • 내장 객체

        내장 객체는 자바스크립트 엔진에 내장되어 있어, 필요한 경우 생성해 사용할 수 있습니다.
        내장 객체로는 문자(String), 날짜(Date), 배열(Array), 수학(Math) 객체 등이 있습니다.

      • 브라우저 객체 모델(BOM)

        브라우저에 계층적으로 내장되어 있는 객체들을 BOM(Browser Object Model)이라고 합니다.
        BOM으로는 window(브라우저), screen, location(주소 창), history, navigator, document(문서 영역) 객체 등이 있습니다.
        브라우저(window)는 document와 location 객체의 상위 객체입니다.

      • 문서 객체 모델(DOM)

        문서 객체 모델(DOM : Document Object Model)이란, (X)HTML[?] 문서 구조를 말합니다.
        (X)HTML 문서의 기본 구조는 최상위 객체로 ⟨html⟩이 있으며, 그 하위 객체로는 ⟨head⟩와 ⟨body⟩가 있습니다.
        자바스크립트를 이용해 이미지의 src 속성을 바꾸고자 한다면, 지정된 ⟨img⟩를 선택해 src 속성을 바꿔야 합니다.
        이렇게 문서 객체 모델에서는 (X)HTML의 모든 요소들을 문서 객체로 선택하여 자유롭게 속성을 바꿀 수 있고, 선택한 객체에 원하는 스타일(CSS)를 적용할 수도 있습니다.

        하지만 자바스크립트의 DOM은 IE8 이하 버전에서호환성이 떨어지기 때문에 사용이 힘들다는 단점이 있어, 최근에는 jQuery의 DOM을 많이 사용하며, 이러한 이유로, JS의 DOM은 깊게 다루지 않습니다.

    • 객체의 생성

      객체의 기본형은 다음과 같습니다.

      var | let | const Object_name = {
      'key(with space)' : value
      key : value;

      key: function func_name( param ) {
      . . .
      }
      }

      또한, 필요에 따라 객체 내에 Getter 혹은 Setter 함수를 추가하기도 합니다.

      JS 역시 다른 OOP와 마찬가지로, 객체에 대해 생성자를 지원합니다.
      생성자는 일반적으로 함수의 이름을 대문자로 시작하도록 명명하며, 새로운 객체를 만들 때에는 new 키워드를 앞에 붙여주어야 합니다.
      자바의 생성자와 다른 점은, 객체 내에 생성자가 위치하는 것이 아닌, 함수로서 별도의 생성자가 만들어진다는 것입니다.

    • 클래스

      자바스크립트의 ES6부터는 클래스 문법이 추가되어, 객체 생성자로 구현했던 코드를 C++, Java와 같은 OOP에서 활용하는 방식과 같이, 조금 더 명확하고 깔끔하게 구현할 수 있도록 합니다.
      기본형은 다음과 같습니다.

      class Class_Name {
      constructor(param1, param2, ... ) {
      this.key = value;
      this.key2 = value2;
      ...
      }
      method( param1, ... ) {
      . . .
      }
      }

      클래스 키워드를 이용해 객체를 만들면 prototype이 자동으로 등록되며, 다음과 같이 상속도 손쉽게 할 수 있습니다.

      class Class_Name extends Parent_Name {
      constructor(param1, param2, ...) {
      super( ... );
      }
      }

      상속 할 때에는 extends 키워드를 사용하며, constructor에서 사용하는 super( ) 함수가 상속받은 클래스(부모 클래스)의 생성자를 가르킵니다.

  3. 내장 객체

    내장 객체(Built-in Object)는 브라우저의 자바스크립트 엔진에 내장된 객체를 말하며, 필요한 경우 객체를 생성해 사용할 수 있습니다.
    내장 객체로는 문자(String), 날짜(Date), 배열(Array), 수학(math), 정규 표현 객체(RegExp Object)등이 있습니다.

    • 내장 객체 생성하기

      new 키워드와 생성 함수를 이용하여 다음과 같이 생성합니다.

      참조 변수(인스턴스 네임) = new 생성 함수( )

      객체의 속성과 메소드는 객체를 생성한 뒤, 참조 변수를 이용하여 다음과 같이 생성합니다.

      참조 변수(인스턴스 네임).변수명 = 값; /* 객체에 속성 할당 */
      참조 변수(인스턴스 네임).함수명 = function() { . . . }; /*객체에 메소드 할당 */

      일반적으로 내장 객체는 new 키워와 생성 함수를 사용해 생성하나, 내장 객체 중의 일부는 new 키워드와 생성 함수를 사용하지 않고, 값만으로 객체를 생성하는 객체들이 있습니다.
      내장 객체를 생성할 때 반듯 ㅣ생성자 함수를 사용해야 하는 경우와 그렇지 않은 경우는 다음과 같습니다.

    • 숫자 객체

      숫자 객체(Number Object)는 JS를 이용하여 숫자를 표기할 때, 표현 가능한 수의 속성과 숫자 표기법에 대한 메서드를 제공합니다.
      숫자 객체는 new 키워드와 생성자 함수를 이용해 객체를 생성할 수 있고, 참조 변수에 값만 입력해도 객체가 생성됩니다.

      let 참조 변수 = new Number(값); or
      let 참조 변수 = 값;

      Number.속성;

    • 수학 객체

      자바스크립트 내장 객체에는 수학과 관련한 기능과 소겅을 제공하는 수학 객체(Math Object)가 있습니다.

      Math.sqrt(숫자)숫자의 제곱근 값을 반환합니다.
      Math.PI(숫자)원주율 상수를 반환합니다.

    • 문자 객체

      문자 객체(String Object)는 문자형 데이터를 객체로 취급하는 것으로, 자바스크립트에서 가장 많이 사용합니다. 문자 객체는 마찬가지로 new 키워드와 문자 객체 함수 String메서드를 사용합니다.

      let 참조 변수=new String(문자형 데이터);

      문자 객체는 다음과 같이 참조 변수에 문자형 데이터만 입력해도 객체가 생성됩니다.

      let 참조 변수=문자형 데이터;

      문자 객체는 문자와 관련된 메서드와 속성을 제공하며, 이를 이용해 문자에 대한 다양한 정보를 구해올 수 있습니다.

      종류설명
      charAt(index)문자열에서 인덱스 번호에 해당하는 문자를 반환합니다.
      indexOf("찾을 문자")문자열에서 왼쪽부터 찾을 문자와 일치하는 문자를 찾아 최초로 일치하는 문자의 인덱스 번호를 반환합니다.
      만일 찾는 문자가 없으면 -1을 반환합니다.
      lastIndexOf("찾을 문자")문자열에서 오른쪽부터 찾을 문자와 일치하는 문자를 찾아 최초로 일치하는 문자의 인덱스 번호를 반환합니다.
      만일 찾는 문자가 없으면 -1을 반환합니다.
      match("찾을 문자")문자열에서 왼쪽부터 찾을 문자와 일치하는 문자를 찾아 최초로 찾은 문자를 반환합니다.
      만일 찾는 문자가 없으면 null을 반환합니다.
      replace("바꿀 문자", "새 문자")문자열에서 왼쪽부터 바꿀 문자와 일치하는 문자를 찾아 최초로 찾은 문자를 새 문자로 치환합니다.
      search("찾을 문자")문자열에서 왼쪽부터 찾을 문자와 일치하는 문자를 찾아 최초로 일치하는 인덱스 번호를 반환합니다.
      slice(a, b)문자열에서 (a+1)~(b+1)번째의 글자를 제외한 나머지를 없앤 뒤 반환합니다.
      음수가 들어올 시 뒤에서부터 기산합니다.
      substring(a, b)a인덱스부터 b인덱스 이전 구간의 문자를 반환합니다.
      음수가 들어올 시 뒤에서부터 기산합니다.
      substr(a, 문자 개수)문자열에 a인덱스부터 지정한 문자 개수만큼 잘라 문자열을 반환합니다.
      split("구분자")구분자를 기준으로 나누어 각 문자열을 배열에 나누어 저장해 반환합니다.
      toLowerCase()문자열에서 영문 대문자를 모두 소문자로 변환합니다.
      toUpperCase()문자열에서 영문 소문자를 모두 대문자로 변환합니다.
      length문자열에서 문자의 개수를 반환합니다.
      concat("문자열")기존 문자열 뒤에 문자열을 이어서 붙입니다.
      charCodeAt(index)문자열에서 인덱스 번호에 해당하는 문자의 아스키 코드를 반환합니다.
      fromCharCode("아스키 코드 값")아스키 코드 값에 해당하는 문자를 반환합니다.
      trim( )문자열 앞뒤의 공백 문자열을 제거합니다.

    • 정규 표현 객체

      정규 표현식(RegExp)를 사용할 수 있는 객체입니다.
      객체에 정규식을 입력한 뒤, 메서드를 이용해 일치 여부를 판단합니다.
      정규 표현 객체를 생성하는 기본형은 다음과 같습니다.

      let 참조 변수 = new RegExp(패턴, 검색 옵션); or
      let 참조 변수 = /패턴/검색 옵션

      정규 표현 객체에서 사용 가능한 메서드는 다음과 같습니다.

      • test( ) : 지정된 규칙에 맞는 단어가 있으면 true 값을 반환합니다.
      • exec( ) : 지정된 규칙에 맞는 단어가 있으면 해당 단어를 반환합니다.

  4. 브라우저 객체 모델(BOM)
    • 브라우저 객체란?

      브라우저에 내장된 객체를 '브라우저 객체'라 합니다. window는 브라우저 객체의 최상위 객체이며, window 객체에 하위 구조를 내포하는 계층적 구조로 구성됩니다.
      그리고 이를 일컬어 브라우저 객체 모델(BOM:Browser Object Model)이라고 합니다.

      window
      document
      screen
      location
      history
      navigator
      body, div, img, a, form, input... 등

    • window 객체

      window는 브라우저 객체의 최상위 객체이며, 다음과 같은 메서드를 사용할 수 있습니다.

      • open( ) : 새 창을 열 때 사용합니다.
      • alert( ) : 경고 창을 띄웁니다.
      • prompt( ) : 질의응답 창을 띄웁니다.
      • confirm( ) : 확인/취소 창을 띄웁니다.
      • moveTo( ) : 창의 크기를 이동시킬 때 사용합니다.
      • resizeTo( ) : 창의 크기를 변경시킬 때 사용합니다.
      • setInterval( ) : 일정 간격으로 지속적으로 실행문을 실행시킬 때 사용합니다.
      • setTimeout( ) : 일정 간격으로 한 번만 실행문을 실행시킬 때 사용합니다.

      각 속성들에 대한 자세한 설명은 다음과 같습니다.

      • open( )

        open( )메서드는 새 브라우저 창을 띄울 때 사용합니다.
        사이트에서 팝업 창을 띄울 때 주로 사용되는 window 객체 메서드로, 기본형은 다음과 같습니다.

        window.open("url 경로", "창 이름", "옵션 설정");

        옵션에서는 팝업 창의 크기, 높이, 위치, 도구상자 노출 여부 등을 설정합니다.
        open( ) 메서드의 옵션 설정에는 다음과 같은 속성들을 설정할 수 있습니다.

        • width : 창의 너비를 설정합니다.
        • height : 창의 높이를 설정합니다.
        • left : 창의 좌/우 수평 위치를 설정합니다.
        • top : 창의 상/하 수직 위치를 설정합니다.
        • location : 창의 url 주소 입력 영역의 노출 여부를 결정합니다. (=yes / =no)
        • status : 창의 상태 영역 노출 여부를 결정합니다. (=yes / =no)
        • scrollbors : 창의 스크롤바의 노출 여부를 결정합니다. (=yes / =no)
        • toolbars : 창의 도구상자 영역의 노출 여부를 결정합니다. (=yes / =no)

      • alert( )

        altert( ) 메서드는 경고 창을 띄울 때 사용합니다.
        window.alert( ) 메서드는 다음 기본형처럼 window 객체를 따로 작성하지 않아도 사용할 수 있습니다.

        alert("경고 메시지");

        경고 창은 코드의 흐름을 중단시켜 '확인'버튼을 클릭해야 alert 메서드의 다음에 나오는 코드를 실행시킬 수 있도록 합니다.

      • prompt( )

        promt( ) 메서드는 질의응답 창을 띄울 때 사용합니다.
        window.prompt( )메서드도 마찬가지로 앞에 window 객체를 따로 작성하지 않아도 됩니다.

        prompt("질의 내용", "기본 답변(placeholder)");

      • confirm( )

        confirm( ) 메서드는 확인/취소 창을 띄울 때 사용합니다. window.confirm( )메서드도 마찬가지로 앞에 window 객체를 따로 작성하지 않아도 됩니다.

        confirm("질의 내용");

      • moveTo( )

        moveTo( ) 메서드는 브라우저 창의 위치를 이동시킬 때 사용합니다. 단, 크롬, 오페라 브라우저에서는 정상적으로 작동하지 않습니다.
        moveTo( ) 메서드는 다음 기본형처럼 스크린(모니터) 왼쪽을 기준으로, x의 위칫값과 위쪽을 기준으로 y의 위칫값을 입력합니다.

        moveTo(x, y);

      • resizeTo( )

        moveTo( ) 메서드는 브라우저 창의 너비와 높이를 바꿀 때 사용합니다. 단, 크롬, 오페라 브라우저에서는 정상적으로 작동하지 않습니다.
        moveTo( ) 메서드는 다음 기본형처럼 스크린(모니터) 왼쪽을 기준으로, x의 위칫값과 위쪽을 기준으로 y의 위칫값을 입력합니다.

        resizeTo(width, height);

      • setTimeout( ) / clearTimeout( )

        setTimeout( ) 메서드는 일정한 간격으로 스크립트 실행문을 한 번만 실행시킵니다.
        하지만 setTimeout( ) 메서드도 재귀 함수 호출을 이용하면 setInterval( ) 메서드처럼 사용할 수 있습니다.
        clearTimeout( ) 메서드는 setTimeout( ) 메서드를 사용하여 일정한 시간 간격으로 한 번 실행할 예정인 실행문을 취소하고자 할 때 사용합니다.
        (링크는 제이쿼리 완성 후에)

        setTimeout( ) 메서드의 기본형은 다음과 같으며, 단위는 ms입니다.

        let 참조 변수=setTimeout("스크립트 실행문", 시간);

        clearTimeout( ) 메서드의 기본형은 다음과 같습니다. 이 때 참조는 앞의 setTimeout( ) 실행문을 참조하는 변수입니다.

        clearTimeout(참조 변수);

    • screen 객체

      screen 객체는 모니터의 너비나 높이와 같은 사용자의 모니터 정보(속성)를 제공하는 객체입니다.

      screen.속성;

      screen 객체에서 사용할 수 있는 속성 값들은 다음과 같습니다.

      • screen.width : 화면의 너비를 반환합니다.
      • screen.height : 화면의 높이를 반환합니다.
      • screen.availWidth : 작업 표시줄을 제외한 화면의 너비값을 반환합니다.
      • screen.availHeight : 작업 표시줄을 제외한 화면의 높이값을 반환합니다.
      • screen.colorDepth : 사용자 모니터가 표현 가능한 컬러 bit수를 반환합니다.

    • location 객체

      location 객체는 사용자 브라우저의 주소 창에 url에 대한 정보(속성)와 새로 고침 기능(메서드)을 제공하는 객체입니다.
      location 객체를 로컬에서 사용할 경우 href 속성만 지원됩니다.

      location.속성 or
      location.메서드( )

      location 객체에서 사용할 수 있는 속성과 메서드는 다음과 같습니다.

      • location.href : 주소 영역에 참조 주소를 설정하거나 URL을 반환합니다.
      • location.hash : URL에 명시된 해시값(#)을 반환합니다.
      • location.hostname : URL에 호스트 이름을 설정하거나 반환합니다.
      • location.host : URL에 명시된 호스트 이름과 포트 번호를 반환합니다.
      • location.port : URL에 명시된 포트 번호를 반환합니다.
      • location.protocol : URL에 명시된 프로토콜을 반환합니다.
      • location.search : URL에 명시된 쿼리(?)를 반환합니다.
      • location.reload( ) : 페이지를 새로 고침 합니다.

      다음은 location 객체를 이용하여 각 정보를 표기한 모습입니다.

    • history 객체

      history 객체는 사용자의 방문 기록과 관련하여 페이지를 조작할 수 있도록 합니다.

      history.메서드; or history.속성;

      history 객체에서 사용 가능한 메서드와 속성은 다음과 같습니다.

      • history.back( ) : 이전에 방문한 페이지로 이동(뒤로가기)합니다
      • history.foward( ) : 다음 페이지로 이동(앞으로)합니다.
      • history.go(n) : n만큼 페이지를 앞으로 혹은 뒤로 이동시킵니다.
      • length : 방문 기록에 저장된 목록의 개수를 반환합니다.

    • navigator 객체

      navigator 객체는 현재 방문자가 사용하는 브라우저와 운영체제의 정보를 제공하는 객체입니다.

      navigator.속성;

      navigator 객체에서 제공하는 속성은 다음과 같습니다.

      • navigator.appCodeName : 방문자의 브라우저 코드명을 반환합니다 :
      • navigator.appName : 방문자의 브라우저 이름을 반환합니다 :
      • navigator.appVersion : 방문자의 브라우저 버전 정보를 반환합니다
      • navigator.language : 방문자의 브라우저 사용 언어를 반환합니다 :
      • navigator.product : 방문자의 브라우저 사용 엔진 이름을 반환합니다 :
      • navigator.platform : 방문자의 브라우저를 실행하는 운영체제를 반환합니다 :
      • navigator.userAgent : 방문자의 브라우저와 운영체제 종합 정보를 제공합니다

  5. 문서 객체 모델(DOM)
    • 문서 객체 모델

      (X)HTML[?] 문서의 구조를 가리켜 문서 객체 모델(DOM: Document Object Model)이라 합니다.
      ⟨img⟩ 태그가 이미지를 출력하는 기능과, src/alt 속성이 있는것처럼, (X)HTML에는 각 태그마다 기능과 속성이 있습니다.
      이렇게 DOM을 사용하면 자바스크립트를 사용하여 문서 객체를 선택하고 속성이나 스타일(CSS)을 적용할 수 있습니다.

      그러나 자바스크립트 DOM은 IE 8 이하 버전과 그 외 파이어폭스, 크롬, 사파리 등 브라우저 간 호환성이 떨어진다는 단점이 있어, 근래에는 JS DOM 대신 jQuery를 주로 사용합니다. DOM은 태그로 이루어지기 때문에, 문서 객체의 구조는 곧 HTML 태그 아래에서 나타나는 태그의 트리적 계층 구조도와 정확히 일치합니다.
      이렇게 HTMl 태그를 최상위 노드로 한 문서 객체 구조의 노드로는 요소 노드(Element node), 텍스트 노드(Text node), 속성 노드(Attribute Node)가 있습니다.

    • 선택자

      (X)HTML의 CSS와 JS의 스타일은 본질적으로 (X)HTML의 요소의 디지인 속성을 바꿀 때 사용한다는 점에서 같습니다.
      그러나, CSS에서는 요소에 특정 동작(이벤트)가 발생한 경우 스타일을 변경시키는 등, 디자인 속성의 가변적 변환이 불가능합니다.
      따라서 (X)HTML의 스타일은 정적이라 표현하고, 자바스크립트 내에서 작성한 스타일은 동적이라고 표현합니다.

      선택자에는 크게 문서에서 요소를 직접 선택해 오는 직접 선택자가 있고, 이 선택자를 사용해 선택해 온 문서 객체를 기준으로 가까이에 있는 요소를 선택하는 인접 관계 선택자가 있습니다.
      직접 선택자에는 id, class, 폼 명(Form Name), 요소 명(Element Name) 선택자 등이 있고, 인접 관계 선택자에는 parentNode, childNodes, firstChild, children, previousSibling, nextSibling 선택자 등이 있습니다.

      선택자를 적용할 때 ⟨head⟩영역에 자바스크립트를 선언하게 되면, (X)HTML을 읽어오는(Loading) 것보다 스크립트의 선택자가 먼저 실행되어 문서 객체를 선택할 수 없게 됩니다.
      또한, (X)HTML 문단 태그보다 먼저 자바스크립트가 문서 상단에 위치해 있어 먼저 실행되면 다음에 오는 문단 태그를 스크립트의 선택자를 사용하여 선택해 올 수 없게 됩니다.
      그러므로 스크립트에 선택문을 사용할 경우애는 ⟨body⟩태그 영역에 문단 태그 작성을 마친 후 마지막 부분에 작성하거나, window.onload=function(){ ... }를 이용하여 선택문을 작성해야 합니다.

    • 문서 객체에 CSS 적용법

      직접 선택자로 문서 객체의 스타일(CSS)을 적용하는 방법은 다음과 같습니다.

      document.직접 선택자(선택 메서드).style.속성=값;

      인접 관계 선택자를 사용하려면 직접 선택자를 이용해 우선 요소를 선택한 후, 해당 요소를 기준으로 인접 관계 선택자를 사용해야 합니다.

      document.직접 선택자(선택 메서드).인접 관계 선택자.style.속성=값;

      자바스크립트에서 CSS 속성을 작성할 때 주의해야 할 점은, (X)HTML의 background-color와 같은 속성을 사용하고자 하는 경우, 자바스크립트에서 하이픈('-')문자를 사용하게 되면 산술 연산자로 인식되므로, backgroundColor와 같이 카멜 케이스로 표기합니다.
      또한, 인접 관계 선택자를 사용할 경우, IE8 이하 버전과 그 밖의 브라우저에서는 결과가 모두 다르게 나오는데, 이는 공백 문자의 인식 여부 차이 때문입니다.

    • 노드 타입으로 인접 관계 선택자 문제 해결하기

      노드 타입(nodeType)을 이용하면 DOM에서 인접 관계 선택자를 이용함으로서 인해 생기는 호환성 문제를 해결할 수 있습니다.
      (X)HTML 노드에는 (X)HTML 태그를 연결하는 요소 노드(Element Node)와 텍스트를 연결하는 텍스트 노드(Text Node), 그리고 (X)HTML 태그에 속성을 연결해 주는 속성 노드(Attribute Node)가 있습니다.

      선택자로 선택한 요소가 어떤 노드인가에 따라 요소 노드는 1, 속성 노드는 2, 텍스트 노드는 3의 노드 타입 값을 같습니다.
      다음은 문단 태그에서 선택자로 각 노드를 선택해 타입을 검사하는 예문입니다.

      ⟨ a1 href2="http://w3.org"id="m"⟩ W3C3 ⟨/a⟩

      1. document.getElementById("m").nodeType; // 요소 노드 : 1을 반환합니다. '⟨a⟩'
      2. document.getElementById("m").firstChild.nodeType; // 텍스트 노드 : 3을 반환합니다. 'W3C'
      3. document.getElementById("m").getAttributeNode("href").nodeType; // 속성 노드 : 2를 반환합니다. 'href'

      그러나 다음 문단 태그에 인접 관계 선택자를 사용해 요소를 선택할 경우, IE8 이하 버전에서 실행하면 다음과 같은 호환성 문제가 발생할 수 있습니다.

      그 결과, 노드 타입 값을 검사하면 1이 반환됩니다.

      IE8 이하 브라우저

      document.getElementById("wrap").firstChild.nodeType; // 노드 타입 값 1을 반환합니다.
      var p=document.getElementsByTagName("p")[1];
      p.nextSibling.nodeType; // 노드 타입 값 1을 반환합니다.

      그러나, 파이어폭스, 크롬, 사파리 등에서 다음과 같이 인접 관계 선택자를 실행하면, (X)HTML의 코드에 줄 바꿈이 일어날 때마다 공백 문자가 생성되어, ⟨p⟩태그가 아닌 공백 문자가 선택됩니다.
      그 결과, 노드 타입 값을 검사하면 1이 아닌 3이 반환됩니다.

      크롬, 사파리, 오페라 등의 브라우저

      document.getElementById("wrap").firstChild.nodeType; // 공백이 선택되어 노드 타입 값 3을 반환합니다.
      var p=document.getElementsByTagName("p")[1]; p.nextSibling.nodeType; // 노드 타입 값 3을 반환합니다.

      document.getElementById("wrap").children[0].nodeType; var p=document.getElementsByTagName("p")[1];
      var nextObj=p.nextSibling;
      (nextObj.nodeType != 1){
