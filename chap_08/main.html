  1. 챕터 8-1. CSS 박스 모델 ▽

    웹 문서의 내용을 마진과 패딩, 테두리 등을 이용하여 박스 형태로 정의합니다.

    • 블록 레벨 요소와 인라인 레벨 요소

      블록 레벨 요소는 혼자서 한 줄을 전부 차지하며(width : 100%), 따라서 요소의 왼쪽이나 오른쪽에 다른 요소가 올 수 없습니다.
      따라서 width, margin, padding 등을 이용해 크기나 위치를 지정하기 위해선 블록 레벨 요소여야만 합니다.

      반면, 인라인 레벨 요소는 줄을 차지하지 않고, 콘텐츠만큼만 영역을 차지합니다. 따라서 한 줄에 여러 개의 인라인 요소를 표현할 수 있습니다.

      다음은 블록 레벨 요소와 인라인 레벨 요소를 만드는 대표적인 태그입니다.

      종류해당 태그
      블록 레벨 태그p, h1~h6, ul, ol, div, blockquote, from, hr, table, fieldset, address
      인라인 레벨 태그img, object, br, sub, sup, span, input, textarea, label, button

    • 박스 모델(box model) : 박스 형태의 콘텐츠

      ⟨p⟩태그와 같은 대부분의 블록 레벨 요소들은 박스 모델 형태로, 요소 앞뒤로 빈 줄이 생기며 박스 형태를 이룹니다.
      박스 모델의 주요 속성들은 다음과 같습니다.

      • width, height 속성 : 콘텐츠 영역의 크기

        이 두 속성은 콘텐츠 영역의 너비와 높이를 지정합니다. 기본형은 다음과 같습니다.

        width: ⟨크기⟩ | ⟨백분율⟩ | auto
        height: ⟨크기⟩ | ⟨백분율⟩ | auto

        실제 콘텐츠 크기를 계산함에 있어, 실제 콘텐츠의 크기는 width+좌우 패딩 + 좌우 보더로 계산합니다.
        그러나 IE6의 경우, 계산법이 표준과 다르기 때문에, IE 6을 고려한 사이트를 기획한다면 width 값을 지정할 때 주의해야 합니다.

      • display 속성 : 화면 배치 방법 결정하기

        block 요소를 inline 속성으로 바꾸거나, inline 요소를 table 속성으로 바꾸는 등, 해당 요소가 화면에 보이는 방식을 지정합니다.
        기본형은 다음과 같습니다.

        display: none | contents | block | inline | inline-block | table | table-cell 등

        각 속성에 대한 설명은 다음과 같습니다.

        • block : 요소를 블록 레벨로 지정합니다.
        • inline : 요소를 인라인 레벨로 지정합니다.

        • inline-block : inline 요소에 블록 레벨 속성 (float, margin, etc)을 적용하기 위해 요소는 인라인으로, 내용은 블록으로 지정합니다.
        • none : visibility: hidden;과는 다르게 요소의 공간까지 포함하여 요소를 화면에서 아예 표시하지 않습니다.
          반응형 웹에서 모바일 화면에서 표시하고 싶지 않은 요소에 해당 속성을 적용하는 것처럼 사용하기도 합니다.
          visibility 속성에 대한 자세한 정보는 이곳에서 확인할 수 있습니다.
        • inherit : 상위 요소의 display 속성을 상속받습니다.
        • table : 블록 레벨의 표로 만듭니다.
        • inline-table : ⟨tange⟩ 태그를 사용한 것처럼 인라인 레벨의 표를 만듭니다.
        • table-row : ⟨tr⟩ 태그를 사용한 것처럼 인라인 표의 행으로 만듭니다.
        • table-row-group : ⟨tbody⟩ 태그를 사용한 것처럼 표의 행을 그룹으로 만듭니다.
        • table-header-group : ⟨thead⟩ 태그를 사용한 것처럼 표의 헤더 그룹으로 만듭니다.
        • table-footer-group : ⟨tfoot⟩ 태그를 사용한 것처럼 표의 요약 영역 그룹으로 만듭니다.
        • table-column : ⟨col⟩ 태그를 사용한 것처럼 표의 열로 만듭니다.
        • table-column-group : ⟨colgroup⟩ 태그를 사용한 것처럼 표의 열 그룹으로 만듭니다.
        • table-cell : ⟨td⟩나 ⟨tr⟩을 사용한 것처럼 표에서 하나의 셀로 만듭니다.
        • table-caption : ⟨caption⟩ 태그를 사용한 것처럼 표의 캡션을 만듭니다.
        • list-item : ⟨li⟩태그를 사용한 것처럼 기본적인 블록 박스와 표시자 박스를 만듭니다.

  2. 챕터 8-2. 테두리 관련 속성들 ▽

    • border-style 속성 : 테두리 스타일 지정하기

      실선이나 점선, 이중 선 등 테두리의 형태를 지정합니다. 기본값은 none입니다.
      기본형은 다음과 같습니다.

      border-style: none | hidden | dashed | dotted | double | groove | inset | outset | ridge | solid

      각 속성에 대한 설명은 다음과 같으며, border-collapse 속성에 대해서는 이곳에서 자세히 확인할 수 있습니다.

      • none : 기본값으로, 테두리가 나타나지 않습니다. 테두리 표시 영역
      • hidden : 테두리가 나타나지 않으며, border-collapse:collapse일 경우, 다른 테두리도 표시되지 않습니다. 테두리 표시 영역
      • dashed : 테두리를 짧은 선(직선으로 된 점선)으로 표시합니다. 테두리 표시 영역
      • dotted : 테두리를 점선으로 표시합니다. 테두리 표시 영역
      • double : 테두리를 이중선(겹선)으로 표시합니다. 테두리 표시 영역
      • groove : 테두리를 창에 조각한 것처럼 표시합니다. 홈이 파인 듯 입체적으로 보입니다. 테두리 표시 영역
      • inset : border-collapse: separate일 경우, 전체 박스 테두리가 창에 박혀 있는 것처럼 표시되고, 테두리 표시 영역
        border-collapse: collapse일 경우, groove와 똑같이 표시됩니다.테두리 표시 영역
      • outset : border-collapse: separate일 경우, 전체 박스 테두리가 창에서 튀어나온 것처럼 표시되고, 테두리 표시 영역
        border-collapse: collapse일 경우, ridge와 똑같이 표시됩니다. 테두리 표시 영역
      • ridge : 테두리를 창에서 튀어나온 것처럼 표시합니다. 테두리 표시 영역
      • solid : 테두리를 실선으로 표시합니다. 테두리 표시 영역

    • border-width 속성 : 테두리 두께 지정하기

      테두리의 두께를 지정합니다. 속성 값이 2개인 경우 (상하, 좌우) 값으로 지정하고, 속성 값이 4개인 경우 (상,우,하,좌) 순서로 지정합니다.
      border-width 속성을 통해 상하좌우 모든 값을 한 번에 지정하거나, 개별 속성을 이용해 개별 값만 별도로 지정할 수도 있습니다.
      이 순서는 모든 border 속성에 통용되며, 기본형은 다음과 같습니다.

      border-width: ⟨크기⟩ | thin | medium | thick or
      border-( top | right | bottom | left )-width: ⟨크기⟩ | thin | medium | thick

    • border-color 속성 : 테두리 색상 지정하기

      테두리의 색상을 지정합니다. 기본형은 다음과 같습니다.

      border-color: ⟨색상⟩ or
      border-( top | right | bottme | left )-color: ⟨색상⟩

    • border 속성 : 테두리 스타일 묶어 지정하기

      다른 속성과 마찬가지로, border 관련 속성은 border 속성을 이용해 묶어 지정할 수 있습니다.
      다만 네 방향의 테두리 스타일이 같지 않다면, 개별 방향에 대하여 별도의 속성을 지정해야만 합니다.
      기본형은 다음과 같습니다.

      border: ⟨두께⟩ ⟨색상⟩ ⟨스타일⟩ or
      border-( top | right | bottom | left ): ⟨두께⟩ ⟨색상⟩ ⟨스타일⟩

    • border-radius 속성 : 박스 모서리 둥글게 만들기

      박스의 모서리를 ⟨크기⟨의 반지름을 가진 원이 겹친 것처럼 둥글게 처리합니다.
      네 모서리를 한번에 처리하지 않을 경우, 각 모서리를 별도로 지정하는 속성을 이용해야 합니다.
      또한, 모서리의 크기를 2개를 입력한다면, 각각 ⟨가로 크기⟩, ⟩세로 크기⟨의 타원으로 인식합니다.
      border-radius 속성으로 네 모서리를 동시에 처리할 경우에는, 가로 반지름과 세로 반지름 사이에 슬래시(/)를 넣어 구분합니다. 기본형은 다음과 같습니다.

      border-radius: ⟨크기⟩ | ⟨백분율⟩ or
      border-top-( left | right )-radius: ⟨크기⟩ | ⟨백분율⟩ or
      border-bottom-( left | right )-radius: ⟨크기⟩ | ⟨백분율⟩

    • box-shadow 속성 : 요소에 그림자 효과 넣기

      선택한 요소에 그림자 효과를 넣습니다. ⟨그림자 값⟩에서 수평 거리와 수직 거리는 반드시 지정해야 하며, 기타 속성 값은 옵션이므로 필요할 떄만 사용하면 됩니다.
      두 개 이상의 그림자를 사용할 경우, 앞의 그림자부터 차례로 적용됩니다. 기본형은 다음과 같습니다.

      box-shadow : none | ⟨그림자 값⟩ [, ⟨그림자 값⟩]*;
      그림자 값 = ⟨수평 거리⟩ ⟨수직 거리⟩ ⟨흐림 정도⟩ ⟨번짐 정도⟩ ⟨색상⟩ inset

      각 속성에 대한 설명은 다음과 같습니다.

      • 수평 거리 : 그림자의 수평 오프셋 거리(양수/음수)입니다. 필수 속성입니다.
      • 수직 거리 : 그림자의 수직 오프셋 거리(양수/음수)입니다. 필수 속성입니다.
      • 흐림 정도 : 그림자의 blur radius를 지정합니다. 기본 값은 0으로, 음수 값은 사용할 수 없습니다.
      • 번짐 정도 : 그림자의 번지는 정도(양수/음수)를 나타냅니다. 기본 값은 0입니다.
      • 색상 : 그림자의 색상을 지정합니다. 한 가지만 지정할 수도 있고, 공백으로 구분해 여러 개의 색상을 지정할 수도 있습니다.
        필요한 경우에만 사용하는 옵션 값이며, 기본 값은 현재 글자 색입니다.
      • inset : 이 키워드를 함께 표시하면 안쪽 그림자로 그립니다.

  3. 챕터 8-3. 여백을 조절하는 속성들 ▽

    이번 장에서는 margin과 padding을 다루는 속성에 대해 알아봅니다.
    margin과 padding 속성 역시, 두 개를 사용할 경우 상하/좌우, 네 개를 사용할 경우 상/우/하/좌 순서로 패딩을 지정합니다.

    • margin 속성 : 요소 주변 여백 설정하기

      border 바깥쪽의 여백인 margin 값을 설정합니다.
      margin을 설정하지 않았으나, 설정하지 않은 방향으로 margin이 설정된 요소가 있는 경우, 해당 방향의 margin은 해당 방향으로 있는 요소의 margin 값을 공유합니다.
      기본형은 다음과 같습니다.

      margin: ⟨크기⟩ | &lang백분율⟩ | auto or
      margin-( top | right | bottom | left ): ⟨크기⟩ | &lang백분율⟩ | auto

      margin의 속성 중 auto 속성은 display 속성에서 지정한 값에 맞게 적절한 값이 자동으로 설정됩니다.
      또한, margin-left와 margin-right를 auto로 지정하면, 요소의 너비 값을 뺀 나머지 공간의 좌우 마진을 똑같이 맞춥니다. 이 방법은 웹 요소를 중앙에 배치하려고 할 때 자주 사용합니다.

    • 마진 중첩(margin overlap)현상

      마진 중첩(마진 상쇄) 현상이란, 요소를 세로로 배치하는 경우 마진과 마진이 만날 때 마진이 큰 쪽으로 겹쳐지는 현상을 말합니다.
      이는 여러 요소를 세로로 배치할 때 맨 위의 마진과 맨 아래 마진에 비해 중간에 있는 마진들이 너무 커지는 것을 방지하기 위한 것입니다.
      마진 중첩은 아래 마진과 위 마진이 서로 만날 때 큰 마진 값으로 합쳐지는 것이고, 오른쪽 마진과 왼쪽 마진이 만날 경우에는 중첩되지 않습니다.

    • padding 속성 : 콘텐츠 영역과 테두리 사이 여백 설정하기

      콘텐츠와 border 사이의 여백인 padding 값을 설정합니다. 기본형은 다음과 같습니다.

      padding: ⟨크기⟩ | ⟨백분율⟩ | auto or
      padding-( top | right | bottom | left ): ⟨크기⟩ | ⟨백분율⟩ | auto
