  1. 모바일 기기와 웹 디자인
    • 반응형 웹 디자인

      PC, 태블릿, 스마트폰 등 다양한 화면 크기를 가진 기기로 웹 사이트에 접속하는 엔스크린(N-Screen)을 지원하기 위한 여러 방법 중, 가장 손쉽게 선택할 수 있는 옵션 중 하나는 '뷰포트'를 사용하는 '반응형 웹 디자인'입니다.
      '반응형 웹 디자인(responsive web design)'은 PC용 웹 사이트와 모바일용 사이트를 별도로 만들어야 하는 불편함 없이, 기존 웹 사이트의 내용을 그대로 유지하면서 다양한 화면 크기에 웹 사이트를 표시할 수 있도록 합니다.
      반응형 웹은 모바일 기기에 따라 반응하는 것이 아니라, 모바일 기기에 있는 '웹 브라우저 창의 너비값'에 따라 반응합니다.

    • 반응형 웹의 장점

      반응형 웹의 장점들은 다음과 같습니다.

      1. 스마트 TV, 게임 콘솔 등, 웹 표준을 지원하기만 하면 어떤 스마트 기기에서든지 접속이 가능합니다.
      2. 스마트폰이나 태블릿 등의 가로 모드에 맞추어 레이아웃을 변형해 보여줄 수 있습니다.
      3. 반응형 웹에 사용되는 코드들은 서버쪽 코드가 없고, HTML과 CSS로만 되어 있어 복잡하지 않으며, 사이트가 하나뿐이기 때문에 유지/관리가 쉽습니다.
        다만, 반응형 웹 사이트를 제작하기 위한 기술 중 미디어 쿼리와 플렉스 박스는 최신 웹 표준인 CSS3의 일부이기 때문에, 하위 버전의 브라우저에서는 사용할 수 없다는 단점이 있습니다.

    • 뷰포트란?

      PC 화면과 모바일 화면은 픽셀의 표현 방법이 다르기 때문에 PC에서 보이는 내용을 모바일 기기에서 그대로 표시할 수는 없습니다.
      뷰포트(viewport)란, 스마트폰 화면에서 실제 내용이 표시되는 영역으로, 뷰포트를 지정하면 화면을 접속한 기기의 화면에 맞추어 확대하거나 축소해 표시할 수 있게 됩니다.
      일반적으로, 웹키트(webkit)기반인 모바일 브라우저들의 기본 뷰포트 너비는 980px로, 웹 페에지 너비를 스마트폰용인 320px로 맞추어 제작하더라도, 스마트폰의 모바일 브라우저의 기본 뷰포트 넓이는 980px이므로, 의도와 다르게 작게 표시됩니다.
      따라서, 반응형으로 웹을 디자인할 경우, px와 같은 고정형 단위보다는 em과 같은 가변형 단위를 이용하는 것이 유리합니다. 가변형 단위에 대해서는 이곳에서 확인할 수 있습니다.

    • 뷰포트 지정하기

      뷰포트는 ⟨meta⟩ 태그를 이용해 ⟨head⟩ 태그와 ⟨/head⟩태그 사이에 작성합니다.
      뷰포트 관련 표준 규약은 이곳에서 확인할 수 있으며, 기본형은 다음과 같습니다.

      ⟨meta name="viewport" content="⟨속성1=값1⟩, ⟨속성2=값2⟩, ... "⟩

      뷰포트의 속성과 속성 값은 content 속성을 이용해 지정하는데, content 안에서 사용하는 뷰포트 속성은 다음과 같습니다.

      속성설명사용 가능한 값기본 값
      width뷰포트 너비device-width 또는 크기브라우저 기본 값
      height뷰포트 높이device-height 또는 크기브라우저 기본 값
      user-scalable확대/축소 가능 여부yes 또는 noyes
      initial-scale초기 확대/축소 값1~101
      minimum-scale최소 확대/축소 값0~100.25
      maximum-scale최대 확대/축소 값0~101.6

      가장 많이 이용하는 뷰포트의 형태는, 뷰포트의 너비를 스마트폰 화면 너비에 맞추고, 초기 화면 배율을 1로 지정하는 것으로 다음과 같습니다.

      ⟨meta name="viewport content="width=device-width, initial-scale=1"⟩

  2. 가변 그리드 레이아웃
    • 고정 그리드와 가변 그리드

      웹 사이트의 레이아웃을 정할 때 자주 사용하는 기준이 '그리드 시스템(grid system)'입니다.
      반드시 그리드 시스템을 따를 필요는 없지만, 사이트 전체의 디자인이나 일관성을 유지하는데에는 그리드 시스템이 편리합니다.
      그리드 시스템이란 화면을 몇 개의 칼럼(column)으로 나누어 요소들을 배치하는 것으로, 필요할 때마다 칼럼들을 묶어 배치할 수 있습니다.
      그리드 시스템을 사용하면 화면을 단순하게 만들면서도 규칙적으로 배열하기 때문에 레이아웃을 일관성 있게 유지할 수 있다는 장점이 있어 대부분의 사이트들에서 사용됩니다.
      그리드 시스템에서는 화면 너비를 몇 픽셀로 하는지에 따라 960px로 고정하는 '960 그리드 시스템'이나, 1200px로 고정하는 '1200 그리드 시스템'으로 나누기도 하고, 화면을 몇 개의 칼럼으로 나누는가에 따라 12칼럼 그리드, 16칼럼 그리드, 24칼럼 그리드로도 나누는데 주로 960px의 12칼럼 그리드를 사용합니다.

      그리드 시스템은 화면 너비를 특정 값으로 고정한 뒤 그 안에 표시할 요소들의 너비 값을 지정해 너비가 항상 일정하게 표시되도록 할 수 있기 때문에 PC용 사이트만 만드는 경우 이 방법을 이용하는 것이 편리할 수 있습니다.
      반면, 어떤 기기에서도 동일한 레이아웃을 가지면서 기기의 특성에 맞게 웹 문서가 표현되게 하려면 '가변 그리드 레이아웃(fluid grid layout)' A.K.A 가변 레이아웃(fluid layout)을 사용합니다.
      가변 레이아웃은 문서 안의 각 요소의 너비를 픽셀과 같은 고정 값이 아니라 백분율과 같은 가변 값으로 지정하는 것을 말하므로, 브라우저 너비 값이 바뀔 때마다 웹 요소의 너비 값 또한 바뀌게 됩니다.
      가변 그리드 레이아웃을 사용할 경우, 너비 값이 줄어들면 실제 콘텐츠를 확인하기 불편해지므로, 가능하면 간결한 디자인을 사용하는 것이 좋습니다.

    • 가변 그리드 레이아웃 만들기

      1. 전체를 감싸는 요소 확인하기

        가변 그리드는 웹 콘텐츠 전체를 감싸는 요소의 너비를 기준으로 각 요소의 너비를 계산합니다.
        따라서 콘텐츠 전체를 감싸는 wrapper 등의 요소를 만들어 주어야 합니다.
        주로 컨텐츠 좌우의 여백 등은 이곳에서 width의 너비를 (96% 정도로)설정해 지정합니다.

      2. 각 요소의 너비 값 계산하기

        기준이 되는 요소를 찾았다면 그 요소의 너비 값으로 각 요소의 너비 값을 계산합니다.
        계산식은 일반적으로 다음과 같은 계산식을 이용합니다.

        ( 요소의 너비 / 콘텐츠 전체를 감싸는 요소의 너비 ) * 100

        요소의 너비 값을 %로 지정할 때 소수점 이하 숫자가 많다면 소수점 이하 3~4자리까지 표시하는 것이 좋습니다.
        그리고 요소의 너비 뿐 아니라 패당과 마진도 백분율 값으로 조정하는 것이 자연스럽습니다.

  3. 가변 레이아웃과 가변 요소

    가변 그리드를 사용하는 경우 글꼴과 같이 너비 값을 지닌 웹 요소들의 너비 또한 브라우저의 너비가 바뀔 때 함께 바뀌어야 자연스럽게 표현될 수 있습니다.

    • 가변 글꼴

      텍스트 크기를 픽셀(px) 단위로 지정하면 크기가 고정되기 때문에 화면 크기가 작은 기기에서는 매우 작게 표시됩니다.
      따라서 반응형 웹 디자인을 위해 가변 그리드 레이아웃을 사용할 때는 글자 크기도 유동적으로 바뀌어야 하는데, 이것을 '가변 글꼴'이라고 합니다.
      가장 많이 사용하는 단위는 em과 rem입니다.

      • em 단위

        em 단위는 부모 요소에서 지정한 폰트의 대문자 M의 넓이를 1em으로 지정한 것으로, 일반적으로 1em은 16px입니다.
        이렇게 em 단위로 작성된 글꼴도 실제로 브라우저 창에 표시될 때는 해당 기기에 맞춘 픽셀 크기로 계산되어 표시됩니다.

      • rem 단위

        em 단위는 부모 요소의 글꼴을 기준으로 하기 때문에 중첩된 부모 요소의 글자 크기의 영향을 받습니다.
        즉, em단위를 사용하면 부모 요소의 크기에 따라 자식 요소의 글자 크기가 바뀌기 때문에, em 수치가 계속 달라진다는 단점이 있습니다.
        이러한 단점을 없애기 위해 rem을 사용합니다. rem은 처음부터 기본 크기를 지정하기 때문에 중간에 기본 값이 바뀌지 않습니다.

    • 가변 이미지

      이미지를 가변 이미지(fluid image)로 만들면 창의 너비에 따라 이미지 너비도 조정되도록 설정할 수 있습니다.
      이미지를 가변 이미지로 만들 수 있는 방법들은 다음과 같습니다.

      • CSS 이용하기

        CSS를 이용해 이미지를 가변 레이아웃에 맞게 표시하려면 이미지를 감싸고 있는 부모 요소만큼만 커지거나 작아지도록 max-width 속성 값을 100%로 지정하면 됩니다.
        IE7 이하 버전까지 지원해야 할 경우, max-width 속성을 인식하기 못하기 때문에 width 속성을 이용해 100%로 지정합니다.
        width는 웹 문서에 삽입할 이미지 너비 값이고, max-width는 가변 이미지에서 최대한 표시할 수 있는 이미지 너비 값입니다.

      • ⟨img⟩ 태그와 srcset 속성

        고해상도 이미지를 크기만 줄여 모바일에 표시할 경우 파일 사이즈가 크기 때문에 모바일에서 다운로드 하는 데 시간이 오래 걸립니다.
        또한 텍스트가 포함된 이미지일 경우, 모바일 화면에 맞게 줄여 표시하면 텍스트 내용을 알아보기 힘든 경우도 있습니다.
        그럴 경우, ⟨img⟩ 태그에서 srcset 속성을 이용하면 화면 너비 값이나 픽셀 밀도[?]에 따라 해상도의 이미지 파일을 지정할 수 있습니다.
        기본형은 다음과 같습니다.

        img src="⟨이미지⟩" srcset="⟨이미지1⟩[, ⟨이미지2⟩, ⟨이미지3⟩, ...]" ⟩

        각 이미지들은 화면 너비가 달라질 때 표시할 이미지들을 나열한 것으로, 이미지 파일 경로와 함께 너비 값이나 픽셀 밀도도 함께 표시해야 합니다.
        픽셀 밀도는 다음과 같이 표시할 수 있습니다.

        ⟨img src="images/pencil.jpg" srcset="images/pencil-hd.jpg 2x" alt="색연필 제품 이미지"⟩

      • ⟨picture⟩ 태그와 ⟨source⟩ 태그 : 상황별로 다른 이미지 표시하기

        HTML 5.1에는 ⟨picture⟩ 태그가 표준으로 포함되면서 ⟨picture⟩태그와 ⟨source⟩ 태그를 함께 사용해 화면 해상도뿐만 아니라 화면 너비에 따라 다른 이미지 파일을 표시할 수도 있습니다.
        ⟨picture⟩ 태그는 IE에서는 지원하지 않고 파이어폭스 38이상, 크롬 38이상, 사파리 9.1이상 버전에서 지원됩니다. iOS 사파리와 안드로이드 브라우저에서는 최신 버전인 iOS 사파리 9.3 이상과 안드로이드 브라우저 53 이상에서만 지원됩니다.

        ⟨source⟩ 태그를 사용하는 방법은 ⟨video⟩ 태그나 ⟨audio⟩ 태그에서 ⟨source⟩ 태그를 이용해 여러 조건의 파일을 따로 지정하는 방법과 비슷합니다.
        ⟨source⟩ 태그에서 사용 가능한 속성은 다음과 같습니다.

        • srcset : 이미지 파일의 경로
        • media : srcset에 지정한 이미지를 표시하기 위한 조건(각 속성 값은 챕터 15에서 확인 가능합니다.)
        • type : 파일 유형
        • sizes : 파일의 크기

    • 가변 비디오

      가변 이미지와 마찬가지로 비디오도 화면의 너비가 달라질 때마다 비디오의 너비가 늘어나거나 줄어들 수 있도록 해야 합니다.
      HTML5에서 동영상을 삽입할 때는 기본적으로 ⟨video⟩ 태그를 사용하므로 CSS를 사용해 max-width 속성을 100%로 지정하면 웹 문서 안에서 크기가 적절히 조절됩니다.
