  1. 챕터 4-1. 폼 만들기 ▽

    폼은 프엔에서 만들고, 백엔드는 ASP/PHP/JSP 등을 이용해 DB로 처리한 뒤 결과값만 돌려받는다.

    • ⟨form⟩ 태그는 폼을 만드는 태그로, 기본형은 다음과 같습니다.

      • ⟨ form [속성="속성 값"] ⟩ 여러 폼 요소 ⟨ /form ⟩

      form을 이용하는 방법은, form 태그로 input 태그와 같이 form을 구성하는 하위 태그를 묶은 뒤, form 속성에 백엔드 속성을 연동하는 방법으로 사용합니다.

      form 태그의 속성
      속성 설명
      method 사용자가 입력한 내용들을 서버 쪽 프로그램으로 어떻게 넘겨줄지 지정합니다.
      속성 값 get - 주소 표시줄에 사용자가 입력한 내용이 그대로 드러납니다. 256byte~4096byte까지의 데이터만 서버로 넘길 수 있습니다.
      post - 대부분 이 방식을 사용합니다. 사용자의 입력을 표준 입력(standard input)으로 넘겨주기 때문에 입력 내용의 길이에 제한을 받지 않고 사용자가 입력한 내용이 드러나지 않습니다.
      name 폼의 이름을 지정합니다. 한 문서 안에 여러 개의 ⟨form⟩ 태그가 있을 경우, 폼들을 구분하기 위해 사용합니다.
      action ⟨form⟩ 태그 안의 내용들을 처리해 줄 서버 상의 프로그램을 지정합니다.
      target action 속성에서 지정한 스크립트 파일을 현재 창이 아닌 다른 위치에 열도록 지정합니다.
      + autocomplete 속성 : 자동 완성 기능으로, on/off 설정이 가능하다.


      폼 태그의 하위 태그

      • ⟨input⟩태그는 폼을 통해 무엇인가를 입력받고자 할 때 이용됩니다. (버튼 혹은 입력창 생성)

        • input 태그에 대해 자세한 사항은 이곳에서 확인할 수 있습니다.

    • ⟨label⟩태그는 폼 요소에 레이블을 붙일 수 있다.



      • 레이블의 기본형은 아래와 같다.

      ⟨ label [속성="속성 값"] ⟩ 레이블 ⟨ /label ⟩ 또는

      ⟨ label for="id이름" ⟩ 레이블 ⟨ /label ⟩
      ⟨ input id="id이름" [속성 = "속성 값"]⟩


      label 태그에서 주의해야 할 점은, label 태그는 해당 input과 연동되어 있어, 라벨 텍스트를 클릭한 경우에도 폼을 클릭한 것과 마찬가지로 취급됩니다.
      따라서, label 태그는 라디오버튼과 같이 사용하면 좋은 조합이 될 수 있습니다.

    • ⟨fieldset⟩태그는 ⟨legend⟩ 태그와 묶어 사용할 수 있습니다.
      fieldset 태그는 태그 안의 폼을 하나의 영역으로 테두리를 그어 묶어 주고, legend 태그는 해당 그룹에 제목을 붙여줍니다.

    챕터 4-2. input 태그 상세 ▽

    input 태그는 type 속성을 이용해 인풋박스의 유형을 바꾸며, 중복 이용되는 경우가 많으므로, id 속성을 이용해 폼 구성 요소를 구분할 것이 권장됩니다.
    ex) ⟨ input type="text" id="user-name" size="10" ⟩

     type 속성의 종류
    • hidden : 사용자에게는 보이지 않지만 서버로 넘겨지는 값을 가집니다.
    • text : 한 줄짜리 텍스트를 입력할 수 있는 텍스트 상자를 넣습니다.
    • search : 검색 상자를 넣습니다. 최신 브라우저의 경우, 'X'키를 통해 검색 기록을 삭제할 수 있습니다.
    • tel : 전화번호 입력 필드를 넣습니다. 사용자 입력 값을 확인하지는 않으나, 해당 입력값이 전화번호임을 인식할 수 있도록 하여, 모바일 접근 시 전화번호 입력 키보드 UI를 제공합니다.
    • url : url 주소를 입력할 수 있는 필드를 넣습니다. 모바일로 접근 시 웹 전용 키보드 UI를 제공합니다.
    • email : 메일 주소를 입력할 수 있는 필드를 넣습니다. 이 필드를 이용하면, 입력한 값이 메일 주소 형식에 맞는지 확인할 수 있습니다.
    • password : 패스워드를 입력할 수 있는 필드를 넣습니다.
    • datetime : 국제 표준시(UTC)로 설정된 날짜와 시간(연, 월, 일, 시, 분, 초, 분할 초)를 넣습니다.
    • datetime-local : 사용자가 있는 지역을 기준으로 날짜와 시간(연, 월, 일, 시, 분, 초, 분할 초)을 넣습니다.
    • date : 사용자 지역을 기준으로 날짜(연, 월, 일)을 넣습니다.
    • month : 사용자 지역을 기준으로 날짜(연, 월)를 넣습니다.
    • week : 사용자 지역을 기준으로 날짜(연, 주)를 넣습니다.
    • time : 사용자 지역을 기준으로 시간(시, 분, 초, 분할 초)을 넣습니다.
    • number : 숫자를 조절할 수 있는 화살표(스핀 박스)를 넣습니다.
    • range : 숫자를 조절할 수 있는 슬라이드 막대를 넣습니다.
    • color : 색상 표를 넣습니다.
    • checkbox : 주어진 항목에서 2개 이상 선택 가능한 체크박스를 넣습니다.
    • radio : 주어진 항목에서 1개만 선택할 수 있는 라디오 버튼을 넣습니다.
    • file : 파일을 첨부할 수 있는 버튼을 넣습니다. 제공되는 UI는 브라우저에 따라서 달라지므로 주의가 필요합니다.
    • submit : 서버 전송 버튼을 넣습니다. 입력된 내용은 form 태그에서 action 속성으로 연결된 프로그램에서 처리됩니다.
    • image : submit 버튼 대신 사용할 이미지를 넣습니다.
    • reset : 리셋 버튼을 넣습니다.
    • button : 버튼을 넣습니다.


    type 속성의 상세

    • hidden필드는 화면상의 폼에는 보이지 않지만, 사용자가 입력을 마치고 폼을 서버로 전송할 때 서버로 함께 전송되는 요소입니다.
      보통 사용자에게 굳이 보여 줄 필요는 없으나, 관리자 입장에서 필요한 것들을 hidden 속성을 이용하여 전달합니다.

      hidden 태그의 기본값은 다음과 같습니다.

      ⟨ input type="hidden" name='이름" value="서버로 넘길 값" ⟩

    • text필드는 한 줄짜리 일반 텍스트를 입력하는 필드입니다.
      폼에서 가장 많이 사용하는 요소로 주로 아이디나 이름, 주소 등 텍스트를 입력할 때 사용하며, 기본형은 다음과 같습니다.

      ⟨ input type="text" [속성="속성 값"] ⟩


      텍스트 필드에서 사용할 수 있는 속성은 다음과 같습니다.
      • name : 텍스트 필드를 구별할 수 있도록 이름을 붙입니다.
      • size :텍스트 필드의 길이를 지정하여 화면에 몇 글자가 보이도록 할 것인지를 지정합니다.
      • value : 텍스트 필드 요소가 화면에 표시될 때 텍스트 필드 부분에 표시될 내용입니다.
      • maxlength : 텍스트 필드에 입력할 수 있는 최대 문자 개수를 지정합니다.

    • search, url, email, tel과 같은 태그들은, text 필드에서 각 정보에 맞추어 분화된 필드 타입입니다.
      따라서, 이들은 모두 텍스트 필드와 같은 속성을 사용한다는 특징이 있습니다.

    • range필드는 슬라이드 막대를 움직여 숫자 값을 입력하게 하나, 일부 브라우저에서는 일반 텍스트 상자로 표시될 수 있습니다.

      range 태그의 기본값은 다음과 같습니다.

      ⟨ input type="range" [속성="속성 값"] ⟩


      레인지 필드에서 사용할 수 있는 속성은 다음과 같습니다.
      • min : 필드에 입력할 수 있는 최솟값을 지정합니다. 기본값은 0입니다.
      • max : 필드에 입력할 수 있는 최댓값을 지정합니다. 기본값은 100입니다.
      • step : 짝수나 홀수 등, 특정 숫자로 제한하고자 할 때 숫자 간격을 지정할 수 있습니다. 기본값은 1이며, 생략할 수 있습니다.
      • value : 필드에 표시할 초기값입니다.

    • radio필드와 checkbox필드는 선택 가능한 항목의 갯수가 다르다는 점만 빼면, 사용법이 비슷합니다.
      radio 태그와 checkbox 태그의 기본값은 다음과 같습니다.

      ⟨ input type="radio | checkbox" [속성="속성 값] ⟩


      라디오버튼과 체크박스에서 사용할 수 있는 속성은 다음과 같습니다.
      • name : 라디오버튼이나 체크박스가 여러 개 있을 경우, 서버의 폼 프로그램에서 라디오버튼이나 체크박스를 구분하기 위해 이름을 지정합니다.
        라디오 버튼은 여러 개 중에서 하나만 선택하는 것이기 때문에 관련있는 그룹끼리는 name 속성을 똑같이 지정해야 합니다.
      • value : 선택한 라디오 버튼이나 체크박스를 서버로 알려 줄 때 넘길 값을 지정합니다. 이 값은 영문이거나 숫자여야 하며 필수 속성입니다.
      • checked : 라디오 버튼의 항목들은 처음에 아무것도 선택되지 않은 상태로 화면에 표시되는데, 기본으로 선택해 놓을 항목이 있다면 checked 속성을 사용해야 합니다.

    • color필드는 js를 이용할 필요 없이 사용자가 선택 가능한 색상표를 제공합니다. 기본형은 다음과 같습니다.

      ⟨ input type="color" [value="기본 색" [속성="속성 값"] ⟩

    • date, month, week는 날짜 유형을 표시하기 위한 필드입니다.
      브라우저 및 플랫폼별로 입력 형식 및 입력 UI는 다르게 적용될 수 있으며, 지원되지 않는 경우에는 텍스트 필드가 표시됩니다.

      기본형은 아래와 같습니다.

      ⟨ input type="date | month | week" [value="기본 값" 속성="속성 값"] ⟩

    • time, datetime, datetime-local은 날짜 및 시간 유형을 지정할 수 있는 필드입니다.
      시간만 지정할 경우 time을 사용하고, 날짜와 시간을 모두 지정하려면 datetime 계열 필드를 사용합니다.
      type="datetime"일 경우, 날짜와 시간 문자 뒤에 timezone을 함께 표시해야 합니다.
      UTC 기준일 경우 Z라고 기입하면 되며, +dd:dd나 -dd:dd처럼 시간의 차이를 직접 기입해도 됩니다.
      기본형은 아래와 같습니다.

      ⟨ input type="time | datetime | datetime-local" [value="기본 값" 속성="속성 값] ⟩


      시간 유형 필드에서 공통으로 사용할 수 있는 속성들은 다음과 같습니다.
      • min : 날짜나 시간의 최솟값을 지정합니다.
      • max : 날짜나 시간의 최댓값을 지정합니다.
      • step : 스핀 박스의 화살표를 누를 때마다 날짜나 시간을 얼마나 조절할지를 지정합니다.
      • value : 화면에 표시할 초기값을 지정합니다. type="time"일 경우, 시간은 00:00부터 23:59까지 입력하고, type="datetime" 계열일 경우 날짜 다음에 키워드 T를 쓰고 24시간제로 시간을 지정합니다.

    • image필드는 버튼 대신 이미지를 사용할 수 있습니다. 기본형은 다음과 같습니다.

      ⟨ input type="image" src="경로" alt="대체 텍스트" [속성="속성 값"] ⟩

    • button필드는 기능이 없는 버튼이기 때문에, onclick 속성을 이용해 스크립트 함수 등을 연결해주어야만 합니다.

      기본형은 다음과 같습니다.

      ⟨ input type="button" [value="버튼 내용"] [속성="속성 값"] ⟩

    챕터 4-3. input 태그의 다양한 속성 ▽

    이번 절에서는 input 태그에 대한 상세 속성들을 확인합니다.

    • autofocus 속성은 페이지가 자바스크립트를 이용할 필요 없이 페이지가 로드된 직후 텍스트 필드에 대해 오토포커싱 될 수 있도록 합니다.
      ex ) ⟨ input type="text" id="uname" autofocus required ⟩
    • placeholder 속성은 텍스트 필드에 대해 플레이스홀더를 제공합니다.
      ex ) ⟨ input type="text" id="uid" placeholder="하이픈없이 입력" ⟩
    • readonly 속성은 텍스트 필드를 읽기 전용으로 지정합니다.
      readonly 속성은 bool type이기 때문에 readonly="true | false"와 같이 지정해주어야 합니다.
      또 다른 사용법으로, readonly="readonly"라고 지정하거나, readonly만 이용할 경우 readonly="true"와 같이 인식합니다.
    • required 속성은 해당 필드를 필수적으로 입력해야만 하는 필드로 지정합니다.
      필수 필드는 브라우저에서 직접 체크하는 것이므로, 오류 메시지 내용은 브라우저들마다 다르게 나타납니다.
      required 속성은 bool type이기 때문에 required="required", 또는 required라고만 입력하면 됩니다.
    • min, max, step 속성은 각각 해당 필드의 최대, 최소값, 그리고 숫자가 변화하는 간격을 지정합니다.
      해당 옵션은 input 태그의 유형이 date이거나 datetime, datetime-local, month, week, time, number, range일 경우 사용 가능합니다.
    • size, minlength, maxlength 속성은 텍스트의 길이를 조절하는 속성입니다.
      size속성은 한 줄짜리 텍스트 필드에서 화면에 보이는 글자의 최대 수를 지정합니다.
      minlength, maxlength 속성은 사용자가 입력해야 할 최대/최소 글자수를 지정합니다.
    • formaction 속성은 실행할 프로그램을 연결합니다.
      type="submit"이나 type="image"일 때 사용할 수 있습니다.
    • formenctype 속성은 서버로 폼을 전송했을 때 폼 데이터를 어떤 방식으로 해석할 것인지 지정합니다.
      type="submit"이나 type="image"일 때 사용할 수 있습니다.
    • formmethod 속성은 서버로 폼을 전송하는 방식(get, post 등)을 지정합니다.
      이미 form 태그 안에서 지정한 방식이 있어도 그 방식은 무시됩니다.
    • formnovalidate 속성은 form 태그 안에 novalidate라는 속성이 있어서 서버로 전송할 때 폼 데이터가 vaidate한지의 여부를 표시할 수 있는데, input 태그 안에서도 formnovalidate 속성을 이용해 유효성 여부를 표시할 수 있습니다.
    • formtarget 속성은 폼 데이터를 서버로 전송한 후 서버의 응답을 어디에 표시할 것인지 타깃을 지정합니다.
    • height, width 속성은 type="image"인 경우 이미지의 너비와 높이를 지정합니다.
    • list 속성은 ⟨datalist⟩에 미리 정의해 놓은 옵션 값을 ⟨input⟩안에 나열해 보여줍니다.
      자세한 내용은 이곳에서 확인할 수 있습니다.
    • multiple 속성은 type="email"이나 type="file"인 경우 두 개 이상의 값을 입력할 수 있도록 합니다.
      이를 이용하려면 input 태그 안에 속성 이름만 표시하면 됩니다.

    챕터 4-4. 여러 데이터 나열해 보여 주기 ▽

    이번 절에서는 input 태그가 아닌 다른 태그를 사용해 한 줄짜리 데이터가 아닌 여러 데이터를 나열해 보여 주고 사용자가 선택할 수 있도록 하는 폼 요소들을 살펴봅니다.


    드롭다운 목록 만들기

    드롭다운 목록은 ⟨select⟩태그와 ⟨option⟩태그를 이용해 표시합니다.
    select 태그로 드롭다운 목록의 범위를 지정하고, 그 안에 option 태그를 사용해 원하는 항목들을 추가합니다.
    마지막으로 option 태그에는 value 속성을 이용해 서버로 넘겨주기 위한 값을 지정하기만 하면 됩니다.

    드롭다운 목록을 만들기 위한 기본형은 다음과 같습니다.

    ⟨ select 속성="속성 값" ⟩
            ⟨ option value ="값" [속성="속성 값"] ⟩ 내용1 ⟨ /option ⟩
            ⟨ option value ="값" [속성="속성 값"] ⟩ 내용1 ⟨ /option ⟩
            ⟨ option value ="값" [속성="속성 값"] ⟩ 내용1 ⟨ /option ⟩
            ...
    ⟨ /select ⟩

    • ⟨select⟩태그를 사용해 만든 드롭다운 목록은 기본적으로 한 가지 옵션이 표시되고, 화살표를 클릭해 나머지 옵션을 살펴본 후 필요한 한 가지 항목을 선택할 수 있습니다.
      이 때, size 속성이나 multiple 속성을 이용하여 드롭다운 목록의 크기나 선택할 항목의 개수를 조절할 수 있습니다.

      • size : 화면에 표시될 드롭다운 메뉴의 항목 개수를 지정합니다. 브라우저에 따라, size에서 지정한 개수+1개의 옵션이 표시되기도 합니다.
        size 0, 혹은 size 지정 안할 경우 크롬에서 size 1과 같이 동작합니다. 다른 브라우저에서도 그런지는 조사가 필요합니다.
      • multiple : 브라우저 화면에 여러 개의 옵션이 함께 표시되면서, Ctrl 키를 누른 상태로 드롭다운 메뉴에 있는 여러 항목을 선택할 수 있게 합니다.

    • ⟨option⟩태그는 드롭다운 목록에 표시되는 옵션들에 대해 지정하는 태그입니다.
      option 태그에서 사용되는 속성들은 다음과 같습니다.

      • value : 옵션을 선택했을 때 서버로 넘겨질 값을 지정합니다.
      • selected : 화면에 표시될 때 기본으로 선택되어 있는 옵션을 지정합니다.

    • ⟨optgroup⟩태그는 드롭다운 목록에서 여러 항목들을 그룹화 할 수 있습니다.
      optgroup 태그를 사용할 때에는 label 속성을 사용해 그룹의 제목을 붙입니다.

    • ⟨datalist⟩태그는 select 태그 대신 이용하며, 이를 통해 텍스트 필드에 드롭다운을 적용할 수 있습니다.
      datalist는 type="text"인 경우 대부분의 브라우저에서 지원하지만, number나 range, color 필드는 브라우저나 버전별로 지원하지 않을 수도 있습니다.
      text 필드를 datalist를 이용하여 구현한 모습은 이러하며, 기본형은 아래와 같습니다.

      ⟨ input type="text" list="datalist_id" ⟩
              ⟨ datalist id="datalist_id" ⟩
                      ⟨ option ⟩ . . . . . . ⟨ /option ⟩
                      ⟨ option ⟩ . . . . . . ⟨ /option ⟩
                      . . . . . .
              ⟨ /datalist ⟩

      datalist의 경우, option 태그에 value 속성과 label 속성을 적용할 수 있습니다.
      각 속성에 대한 설명은 다음과 같으며, 구현된 모습은 이전에 구현된 것과 같습니다.
      • value : 사용자가 레이블을 선택했을 때 서버로 넘겨질 값을 지정합니다.
      • label : 사용자를 위해 브라우저에 표시할 레이블입니다. 따로 지정하지 않을 경우 value 값을 레이블로 사용합니다.

    ⟨textarea⟩ 태그는 input 태그와 달리 여러 줄을 입력할 수 있는 텍스트박스를 제공합니다.
    textarea에서 이용하는 속성은 다음과 같습니다.

    • name : 다른 폼 요소와 구분하기 위해 텍스트 영역의 이름을 지정합니다.
    • cols : 텍스트 영역의 가로 너비를 문자 단위로 지정합니다.
    • rows : 텍스트 영역의 세로 길이를 줄 단위로 지정합니다. 지정한 숫자보다 줄 개수가 많아지면 스크롤 막대가 생깁니다.

    챕터 4-5. 기타 다양한 폼 요소들 ▽

    • ⟨button⟩태그를 이용하면 input 태그의 속성을 이용하는 대신 버튼을 직접 삽입할 수 있습니다.
      input 태그 대신 button 태그를 사용할 경우 그 모양과 역할은 거의 같으나, 화면 낭독기 상에서 해당 요소가 버튼임을 인지할 수 있기 때문에, 접근성 면에서 유리합니다.
      또한, button 태그를 사용한 버튼에는 콘텐츠를 포함할 수 있기 때문에 아이콘을 추가할 수도 있고, CSS를 이용해 원하는 형태로 꾸밀 수도 있습니다. 기본형은 다음과 같습니다.

      ⟨ button [type="submit | reset | button"] ⟩ 내용 ⟨ /button ⟩

      버튼 태그의 속성은 버튼이 활성화되었을 때 어떤 동작을 하는지 지정합니다.
      선택할 수 있는 값은 submit과 reset, button중 하나이고, 기본값은 submit입니다.
      각 속성에 대한 설명은 다음과 같습니다.
      • submit : 폼을 서버로 전송합니다. ⟨ button type="submit" ⟩전송하기⟨ /button⟩ 와 같이 사용합니다.
      • reset : 폼에 입력한 모든 내용을 초기화시킵니다. ⟨ button type="reset" ⟩다시 쓰기⟨ /button ⟩ 와 같이 사용합니다.
      • button : 버튼 형태만 만들 뿐 자체 기능은 없습니다. ⟨ button type="bytton" ⟩주소 입력⟨ /button ⟩ 와 같이 사용합니다.

      • 또한 버튼 태그는 블록 내부에 이미지를 삽입함으로서 이미지를 적용하거나, CSS를 이용해 마우스 오버 시의 스타일도 별도로 지정할 수 있습니다.
      게다가 태그와 CSS만으로 버튼을 만들었기 때문에 버튼을 이미지 파일로 만들어 ⟨input type="image"⟩로 넣었을 때보다 서버의 리소스를 적게 사용합니다.
      다음의 예시는 버튼 태그와 CSS를 통해 버튼을 구현한 예시입니다. 체크 버튼 이외에는 모두 태그와 CSS만을 이용하여 구현하였습니다.

    • ⟨output⟩태그는 입력하는 값이 계산 결과라는 것을 브라우저에게 알려줍니다.
      output 태그를 사용한다고 해서 브라우저 화면에 다르게 표시되는 것은 아니지만, output 태그로 묶인 부분이 일반 텍스트가 아니라 계산의 결과 값이라는 점을 웹 브라우저가 정확히 인식할 수 있습니다.
      이를 이용해 결과 값만 골라 다른 연산을 할 수도 있고, 스크립트 프로그래밍에서 조건 처리에 사용할 수도 있습니다.

      output 태그의 기본형은 다음과 같습니다.

      ⟨ output [속성="속성 값"] ⟩내용⟨ /output ⟩


      다음은 output 태그를 이용하여 덧셈 스크립트를 적용한 예시입니다.
      + =

    • ⟨progress⟩태그는 작업 진행 상태를 나타낼 때 사용하는 태그입니다.
      작업 시작을 0으로 하고 최종 완료를 최댓값으로 해 얼마나 진척되었는지 숫자로 표현합니다.
      이 때 사용하는 값에는 특별한 단위가 없고, 단위를 표시하지도 않습니다.
      progress 태그의 기본형은 다음과 같습니다.

      ⟨ progress value="값" [max="값"] ⟩⟨ /progress ⟩


      progress 태그에서 사용할 수 있는 속성은 다음과 같습니다.
      • value : 작업 진행 상태를 나타내며 부동 소수점으로 표현합니다. 이 값은 0이상, max이하여야 합니다. max가 지정되지 않은 경우 이 값은 1.0보다는 작아야 합니다.
      • max : 작업이 완료되려면 얼마나 많은 작업을 해야 하는지 부동 소수점으로 표현합니다. 이 값은 0보다 커야 합니다.

      다음 예제는 progress 태그를 이용해 진행 상황을 표시한 것입니다.
      자바스크립트를 이용하여 progress 막대의 진행을 순차적으로 진행시켜 애니메이션처럼 동작하도록 하였습니다.

    • ⟨meter⟩태그는 progress 태그와 거의 유사하나, 진행도를 전체 크기에 비례하여 표시하는 특성을 가집니다.
      meter 태그의 기본형은 다음과 같습니다.

      ⟨ meter value="값" [속성="속성 값"] ⟩⟨ /meter ⟩


      meter 태그에서 사용할 수 있는 속성은 다음과 같습니다.
      high값이나 low값이 지정되어 있으면 현재 값이 높은지 낮은지 비교할 수 있으며, 그 정도를 색상으로 표시할 수도 있습니다.

      • min, max : 범위의 최솟값과 최댓값을 나타냅니다. 값을 정하지 않으면 0과 1로 간주합니다.
      • value : 범위 내에서 차지하는 값을 나타냅니다.
      • low : "이 정도면 낮다."라고 할 정도의 값을 지정합니다.
      • high : "이 정도면 높다."라고 할 정도의 값을 지정합니다.
      • optimum : "이 정도면 적당하다."라고 할 정도의 범위를 지정합니다.
        optimum 값이 high값보다 크다면 value값이 클수록 좋고, optimum 값이 low값보다 작다면 value 값이 작을수록 좋습니다.

      아래는 그 예시입니다.
      • 현재 값만 지정한 기본형입니다.
      • 현재 값과 함께 min, max를 지정했습니다.
      • 전체 크기는 1024~10240인데, high값인 8192보다 현재 값(9126)이 더 큽니다.
      • 전체 1중에서 현재 0.5를 차지하고 있으며, 적정도를 0.8로 설정했습니다.
