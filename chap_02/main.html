HTML 태그 정리하기

한 번도 본 적 없던 태그들을 위주로 정리합니다.

(기억이 잘 안 나는 녀석들도 포함합니다.)

  1. 챕터 2-2. 기본 태그 ▽

    • ⟨blockquote⟩ 태그는 블럭으로 구분된 인용 텍스트입니다. ⟨cite⟩ 속성을 이용해 인용 사이트 주소를 표시할 수 있으며, 들여쓰기를 통해 인용이 표현됩니다.
      이것이 ⟨q⟩ 태그와 다른 점은, ⟨q⟩ 태그의 경우 인라인 레벨 태그로 줄바꿈 없이, 따옴표를 이용해 인용임이 표시되지만 ⟨blockquote⟩ 태그의 경우 블록 레벨 태그이기 때문에 블록 구분을 통해 인용 내용이 구분된다는 점입니다. 

    • ⟨pre⟩태그의 경우, 입력하는 그대로 화면에 표시해 주는 기능을 가지고 있습니다.
      
                        js 코드 같은 것들은 그대로 표시할 수 있지만, HTML 태그같은 것은 적용이 되어 표현됩니다.
      
                        ⟨code⟩나 ⟨samp⟩, ⟨kbd⟩ 등 코드와 관련된 내용을 표시할 때 이용하면 좋습니다.
      
                        다만, 웹 접근성 부분에서 좋지 않기 때문에 사용에 있어서 주의가 요구됩니다.
      PRE 태그 사용시 주의 사항 : ▶이미지 펼치기

      pre 태그는 그 특성상, 모바일에서 표시할 때에도 화면 크기를 무시하고 내용이 적힌 대로 표시하게 됩니다. 아마 추후에 화면 레이아웃을 지정할 때에도 이를 어길 가능성이 있다고 추측되므로, 사용 시 주의가 요구됩니다.

    • ⟨strong⟩ 태그는 ⟨b⟩ 태그와 마찬가지로 글자를 굵게 표시합니다.
      ⟨b⟩ 태그와의 차이점은 의미 중심적이라는 것으로, 덕분에 낭독기 접근성이 향상된다는 것입니다.
      (현재 CSS를 통해 strong 태그에 색상 설정이 되어 있는 상태입니다.)

    • ⟨em⟩태그의 경우 ⟨strong⟩태그의 ⟨i⟩talic 버전이라고 이해할 수 있습니다.
      ⟨i⟩태그에 대한 의미론적 버전이라고 할 수 있겠네요.

    • ⟨q⟩태그는 따옴표로 표시되는 인라인 레벨 태그입니다.

    • ⟨mark⟩태그는 형광펜 효과를 내는 태그입니다. CSS의 background-color 속성을 사용해 배경 색상을 바꿀 수 있습니다.

    • ⟨span⟩태그는 일부 영역에만 효과를 주기 위해 이용되는 범위 지정 인라인 태그입니다.
      인라인 대신 단락을 블록 단위로 적용하고 싶다면 ⟨div⟩ 태그를 이용합니다.
    • ⟨ruby⟩ 태그는 간단히 말해, 振ふり仮名がな(후리가나) 전용! 이라고 생각하면 될 듯 합니다.

    • ⟨abbr⟩ 태그는 title 속성을 함께 이용하여 약자를 표시하는데 이용합니다.
      abbr을 이용할 경우 텍스트 아래에 점선이 표시되지만, abbr이 아니더라도 해당 텍스트를 싸고 있는 태그에 타이틀을 적용할 경우, 마우스오버가 된 뒤 잠시 시간이 지나면 태그에 적용된 타이틀이 표시됩니다.
      다만 아직 모든 태그에 적용되는지는 테스트 해 보지 않았습니다.
      또 한 가지 주의할 점은, hierarchy에 따라 해당 오브젝트를 포함하는 상위의 태그에 타이틀이 적용되어 있으며, 해당 오브젝트에 타이틀이 적용되지 않은 경우, 상위 태그의 타이틀을 표시합니다.
      설명이 좀 별로긴하지만 C언어의 지역 변수 전역 변수 비슷한 규칙이라고 생각하면 될 것 같습니다.


    • ⟨cite⟩ 태그는 웹 문서나 포스트에서 참고 내용을 표시합니다. - https://www.w3schools.com/tags/tag_cite.asp

    • ⟨code⟩ 태그는 컴퓨터 인식을 위한 소스 코드를 적용합니다. 오작동 방지를 위해 ⟨pre⟩ 태그와 함께 이용할 것이 권장됩니다.

    • ⟨kbd⟩ 태그는 키보드 입력이나 음성 명령 같은 사용자 입력 내용을 표시합니다. 예를 들어, 현 페이지는 ' F5 ' 키를 누르면 새로고침됩니다.

    • ⟨small⟩ 태그는 부가 정보처럼 작게 표시해도 되는 텍스트를 표시하는 데 이용됩니다.

    • ⟨sub⟩ 태그와 ⟨sup⟩ 태그는 각각 아래 첨자sub와 위 첨자sup를 표현하는 데에 이용됩니다.

    • ⟨s⟩ 태그는 취소선을 긋는 데에 이용됩니다. 나무위키가 생각나면 당신은 이미 진 겁니다. - https://namu.wiki/w/취소선?from=취소선 드립#s-6

    • ⟨u⟩ 태그는 밑줄을 긋는 데에 이용됩니다. 링크인 척 색깔을 넣었지만 클릭해도 아무것도 연결되어 있지 않습니다. 눌러보려 했다면 이번에도 당신은 진 겁니다.

  2. 챕터 2-3. 목록을 만드는 태그 ▽

    목록을 만드는 태그는 여러 가지가 있으며, 그 목록은 아래와 같습니다. A. B. C.와 같은 부분만 강조하려면 스타일시트를 이용하는 방법밖에 없을까요? 한 번 찾아 봅시다.
    1. ul & li
    2. ol & li
    3. dl & dt & dd

    *tip : ⟨/li⟩태그는 생략 가능합니다.

    1. ⟨ul⟩태그와 ⟨li⟩태그는 순서가 없는 목록을 표시하고자 할 때에 이용됩니다.
      각 항목은 원이나 사각형 모양의 불릿(bullet)으로 표시되며, HTML5부터는 CSS의 "list-style-type" 속성을 이용해 불릿의 모양을 수정할 수 있습니다.
      • Unordered List가
      • 사용되는 모습
      • Chapter 02-2에도 li가 적용되었으나
      • 불릿의 hierarchy가 다르기 때문에 불릿이 적용된 색상이나 모양이 다른 것으로 보인다.

    2. ⟨ol⟩태그와 ⟨li⟩태그는 순서가 있는 목록을 표시하고자 할 때에 이용되며, Chapter 02-3에 해당하는 해당 문서 역시, ol을 이용하고 있습니다.
      각 항목은 여러 속성이 함께 사용되어 목록의 숫자 표기 방법이나 시작하는 숫자 등을 바꿀 수 있습니다.
      1. type 속성 : 항목의 머리에 붙는 리스팅용 문자를 변경할 수 있습니다.
        • 숫자 : 기본값으로, 속성 값은 '1'입니다.
        • 영문 소문자 : 속성 값은 'a' 입니다.
        • 영문 대문자 : 속성 값은 'A' 입니다.
        • 로마자 소문자 : 속성 값은 'i' 입니다.
        • 로마자 대문자 : 속성 값은 'I' 입니다.
      2. start 속성 : 리스팅 숫자를 지정한 숫자부터 시작할 수 있게 하는 속성입니다.
      3. reversed 속성 : 항목을 역순으로 표시합니다.

    3. ⟨dl⟩과 ⟨dt⟩, ⟨dd⟩ 태그는 '제목'과'설명'이 한 쌍인 설명 목록(description list)를 만듭니다. 사용 예시는 다음과 같습니다.

      제주 올레
      올레 1코스
      코스 : 시흥 초등학교 옆 - 광치기 해변
      거리 : 14.6km(4~5시간)
      난이도 : 중
      올레 2코스
      코스 : 광치기 해변 - 온평 포구
      거리 : 14.5km(4~5시간)
      난이도 : 중

  3. 챕터 2-4. 표를 만드는 태그 ▽

    표를 만들기 위해서는 기본적으로 ⟨table⟩ 태그로 표를 그릴 영역을 지정한 다음, ⟨tr⟩, ⟨td⟩, ⟨th⟩, 태그를 이용하여 표 안에 열과 행을 지정합니다.
    표는 Row(행), Column(열), Cell(셀)로 구분되며, rowspan, colspan 등의 속성을 이용해 셀을 병합할 수 있습니다.

    테이블 기본 태그

    스타일이나 속성을 제외한, 테이블 기본 조작 태그
    태그 설명
    ⟨table⟩ 테이블 영역을 설정하는 태그로, tr이나 td같은 기본 태그부터, caption이나 thead같은 스타일 지정 태그까지 테이블 영역에 속해야 한다.
    ⟨tr⟩ 테이블에 행 하나를 추가하는 태그로, 데이터가 삽입되지 않았기 때문에 데이터 태그를 넣기 전까지는 형식뿐인 태그이다.
    ⟨td⟩ 테이블 데이터를 입력하는 태그로, 행에 컬럼 하나를 추가하는 기능을 가진다.
    ⟨th⟩ 테이블 헤더를 입력하는 태그로, 태그 내에 입력된 글자를 굵게 하며 가운데 정렬하는 기능을 가진다.
    셀의 병합은 colspan과 rowspan 속성을 이용하며, 기본형은 다음과 같습니다. (td, th 모두 이용 가능합니다)

    <(td/th) colspan="합칠 셀의 개수"> 내용 </(td/th)>
    <(td/th) rowspan="합칠 셀의 개수"> 내용 </(td/th)>

    병합의 기준은 속성을 적용한 셀이며, 기준 셀로부터 1base counting으로 적용하며, 아래와 같이 적용될 수 있습니다.
    ▶셀 병합 예시
    이름 연락처
    주소
    자기소개
    ▲colspan="3"이 적용된 예시

    테이블 형태의 자료 또한, 웹 표준에 맞추기 위해서 다양한 태그들이 사용되며, 표에 제목을 붙이기 위해서는 <caption> 태그와 <figcaption> 태그가 사용됩니다.
    caption 태그의 경우, 상단 표에 위치한 '테이블 기본 태그'에 적용되어 있으며, 표에 대한 설명은 <p>태그를 이용해 적용되어있습니다.
    제목과 설명이 하나의 <caption>태그로 적용되어있습니다.

    <figcaption> 태그는 설명 글을 붙이고 싶은 대상을 figure 태그로 감싼 후, figcaption 태그를 이용해 제목이나 설명 글을 입력하니다.
    이 경우, caption태그는 사용하지 않으며, caption과 달리 중앙에 정렬되지 않습니다.
    표에 대한 설명을 작성할 때에도, 화면 낭독기를 통한 접근성의 향상 방법에 대해서도 고려해 볼 수 있습니다.
    이런 접근성의 향상은, table 태그에 aria-describedby 속성을 적용하는 것을 통해 제공될 수 있습니다. 아래 예제는 figure태그를 이용해 표 전체를 감싼 후, figcaption태그를 이용해 표에 제목을 붙이고, aria-describedby 속성이 적용된 예시입니다.

    테이블 역시 레이아웃에 쓰기 좋게 <thead>,<tbody>,<tfooter>로 머리가슴배 삼단 나누기 기능을 제공합니다.
    이렇게 테이블 구조를 정의해서 사용하면, 다음과 같은 장점들을 얻을 수 있습니다.
    • 화면 낭독기를 통한 접근성이 향상됩니다.
    • CSS를 사용해 정의된 구조 별로 각각 다른 스타일을 적용할 수 있습니다.
    • 특정 구조에만 JS를 적용할 수 있습니다. (tbody에만 스크롤을 적용하는 등의 작업이 가능하겠네요)

    테이블 구조가 적용된 예시의 경우, CSS의 적용을 위해 별도의 페이지로 분리해 두었습니다. (예시 페이지로 이동하기)
    표에서 행 단위로 묶어서 적용을 하기 위해서는 <tr> 태그를 사용하면 됩니다. 하지만 열의 경우 <tr> 태그의 하위에 각각 <td>나 <th>태그로 묶여있기 때문에, 마음껏 선택하기가 힘듭니다.
    때문에 열 단위로 묶어 스타일을 지정하기 위해서는 <col> 태그나 <colgroup> 태그를 이용해야 합니다.

    주의사항 : col과 colgroup 태그는 caption 태그보다는 뒤, tr, td태그보다는 앞에서 사용되어야 합니다.

    • <col> 태그는 한 열에 있는 모든 셀에 같은 스타일을 적용하려고 할 때 사용하는 것으로, 닫는 태그가 없습니다.
      col 태그를 사용해 여러 열을 묶으려면, span 속성을 이용해 단위 셀의 개수를 지정해 주어야 합니다. 예시: <col span="2">
    • <colgroup>태그를 사용해서도 여러 열을 묶어 적용할 수 있는데, colgroup 태그 안에 묶는 열의 개수만큼 col 태그를 넣으면 됩니다.
