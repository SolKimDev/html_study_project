  1. 웹과 멀티미디어

    HTML4까지는 웹 브라우저에서는 멀티미디어를 직접 재생할 수 없었기 떄문에, 별도의 프로그램인 플러그인을 이용했습니다.
    HTML5에서 대부분의 멀티미디어는 브라우저에서 직접 재생할 수 있지만, 호환성 등의 이유로 플러그인은 여전히 유효합니다.
    HTML5에서 사용되는 플러그인 관련 태그는 다음과 같습니다.

    • ⟨object⟩ 태그 : 외부 파일 삽입하기

      object 태그는 웹 므라우저에서 직접 재생할 수 없는 자바 애플릿이나, PDF 파일, 플래시 무비 같은 콘텐츠를 웹 문서 안에 포함시키기 위해 사용합니다.
      이 태그를 이용해 다른 HTML 문서나, 이미지, 오디오, 비디오도 포함시킬 수 있으나, 이런 콘텐츠는 img, audio, video 등 특성화된 태그를 이용합니다.

      기본형은 다음과 같습니다.

      ⟨ object data="경로" type="유형" [name="이름" width="너비" height="높이"] ⟩ 대체 텍스트 ⟨/object⟩

      ⟨img⟩태그에서 src 속성을 반드시 써야 하듯이, 특정 외부 파일을 가져와 표시헐 것인지 여부를 알려 주기 위해 최소한 data나 type 속성 둘 중 하나를 반드시 사용해야 합니다.

    • ⟨embed⟩ 태그 : 외부 파일 삽입하기

      object 태그와 달리 닫는 태그가 없는 embed 태그는 주로 object 태그를 지원하지 않는 이전 브라우저에서 사용됩니다. 기본형은 다음과 같습니다.

      ⟨ embed src="경로" type="유형" width="너비" height="높이" ⟩

    • 멀티미디어의 웹 표준화

      HTML5는 PC뿐 아니라 웹 므라우저가 설치된 모든 기기에 적용되는 웹 표준이므로 플러그인 프로그램 없이 웹 므라우저 자체에서 멀티미디어를 재생할 수 있습니다.
      웹에서 지원되는 비디오/오디오 파일에는 여러 종류가 있습니다.

      종류확장자설명
      비디오mp4고화질 영상을 지원해 많은 사이트에서 사용되고 있습니다. 라이선스가 있지만 웹에서 사용할 경우에는 무료로 사용할 수 있습니다.
      webm화질이 우수하고 무료로 제공되어 최근 많이 사용되고 있습니다.
      ogv화질은 다른 비디오 유형보다 떨어지지만 무료라는 장점 때문에 webm 형식이 등장하기 전 많이 사용되었습니다.
      오디오mp3대부분의 음원에서 사용되지만 라이선스 때문에 유료로 사용해야 합니다.
      ogg무료이므로 게임 등에서 많이 사용합니다.
      아직까지 멀티미디어 파일 형식이 하나의 표준으로 정해진 것은 아니지만, 대부분 mp4/webm, mp3/ogg 파일을 주로 사용합니다.

      다음은 브라우저별 비디오/오디오 파일 지원 여부를 나타낸 표로 괄호 안의 내용은 해당 미디어 파일을 지원하는 버전입니다.

  2. 오디오 & 비디오 재생하기

    HTML 태그를 이용해 페이지 내에서 오디오와 비디오를 재생합니다.

    • ⟨audio⟩ 태그 : 오디오 파일 삽입하기

      HTML5에서 배경 음악이나 효과음 등 오디오를 삽입할 때 사용합니다. 일반적으로 대부분의 브라우저에서 가능하면 mp3 파일을 삽입하며, ogg/oga는 함께 지정하지 않는 경우가 많습니다.
      기본형은 다음과 같습니다.

      ⟨ audio src="오디오 파일 경로" [속성] [속성="속성 값"] ⟩ 대체 텍스트 ⟨/audio⟩

    • ⟨video⟩ 태그 : 비디오 파일 삽입하기

      비디오를 삽입할 때 사용합니다. 기본형은 다음과 같습니다.

      ⟨ video src="비디오 파일 경로" [속성] [속성="속성 값"] ⟩ 대체 텍스트 ⟨/video⟩

    • ⟨source 태그 : 여러 미디어 파일 한꺼번에 지정하기

      브라우저에 따라 지원하는 오디오 코덱이나 비디오 코덱이 다르기 때문에, 한 가지 파일만 사용했을 경우 일부 오래된 브라우저에서는 지원하지 않을 수 있습니다.
      따라서 사용자들의 브라우저 환경을 모두 고려한다면 최신 브라우저와 이전 브라우저에서 모두 재생할 수 있도록 ogv 파일도 함께 지정해 주어야 합니다.
      최근에는 물 코덱을 사용하면서 화질도 뛰어난 webm파일까지 함께 사용합니다.
      브라우저 버전별 지원 상황을 알고 싶다면 이곳에 접속한 후 [Show all] 링크를 클릭해 확인할 수 있습니다.

      이처럼, 여러 형식의 비디오 파일을 함께 사용하려면 video 태그와 source 태그를 함께 사용해 여러 개의 파일을 지정해야 합니다.
      source 태그에는 미디어 파일의 경로를 지정하는 src 속성과 미디어 파일의 형식을 알려주는 type 속성을 사용합니다.
      간단히 type 속성만 상죵할 수도 있고, codecs 속성을 이용해 코덱까지 함께 표시할 수도 있습니다.

      모든 타입의 속성을 이용해 태그를 다음과 같이 기입할 수 있습니다.

      ⟨ source src="video.ogv" type="video/ogg; codecs='theora,vorbis;" ⟩

      비디오 파일을 업로드 할 때에 주의할 점은 ogv, webm 파일 등은 이전에 없던 형식으므로, 웹 서버에서 확장자를 인식할 수 있도록 MIME 속성 유형을 추가해야 할 수도 있습니다.
      또한, 이전 버전의 브라우저에서는 video 태그를 지원하지 않거나, 몇 가지의 확장자를 지원하지 않는 경우도 있으므로, 다음과 같이 embed 태그나 object 태그를 사용해 플러그인으로 삽입할 수 있습니다.

      video 태그에서는 top-down으로 video 태그 안에 있는 source / object / embed 태그 중 한 가지라도 실행되면 나머지는 실행되지 않으므로 위와 같이 이용하여도 중복 실행에 대한 문제는 일어나지 않습니다.
      또다른 방법으로, embed/object 태그를 사용하는 대신, 해당 영역에 'HTML5를 지원하는 브라우저가 필요합니다' 등 대체 텍스트를 넣는 방법도 있을 수 있습니다.

    • ⟨audio⟩ 태그와 ⟨video⟩ 태그의 속성

      ⟨audio⟩ 태그와 ⟨video⟩ 태그에서 사용할 수 있는 속성은 거의 일치하기 때문에 이번 단락에서 한 번에 소개합니다.
      audio / video 태그 한 가지만 사용할 경우, 해당 태그에 src 와 속성을 함께 적용하면 되지만, 여러 확장자를 한 번에 사용할 경우 audio / video 태그에 속성을 넣은 뒤, 그 하위에 source 태그를 넣어 속성을 적용할 수 있습니다.

      각 속성에 대한 설명은 다음과 같습니다.

      • width / height : 둘 중 하나만 지정해도 영상 비율에 따라 자동으로 조정됩니다.
      • autoplay : 오디오/비디오를 자동으로 재생합니다. 크롬의 경우 meted 속성을 사용해 오디오를 제거해야 동영상이 자동으로 재생됩니다.
      • controls : 웹 화면에 재생/멈춤, 진행 바, 볼륨 등이 표시되는 컨트롤 막대 UI를 표시합니다. (웹 브라우저별로 컨트롤 막대의 디자인이 다릅니다)
      • loop : 오디오/비디오를 반복 재생합니다.
      • muted : 오디오/비디오를 재생해 진행하지만 소리는 끕니다.
      • preload : 재생 버튼을 눌러 재생하기 전 오디오/비디오 파일을 미리 다운로드해 준비해 둘 것인지, 혹은 일부 정보만 다운로드할 것인지 여부를 지정합니다.
        사용할 수 있는 속성 값은 다음과 같습니다.
        • none : 미디어 파일을 미리 다운로드하지 않고, 사용자가 재생 버튼을 눌러야만 다운로드하기 시작합니다.
        • metadata : 미디어 파일을 즉시 사용하지 않을 것이라고 생각해 미디어 파일 전체를 다운로드하지 않고 메타 정보만 다운로드합니다.
        • auto : 사용자가 즉시 이용할 수 있도록 웹 문서를 로드할 때 미디어 파일도 모두 다운로드합니다. 다만, 다운로드가 끝나도 사용자가 재생 버튼을 눌러야만 재생됩니다.
          속성 값을 지정하지 않고 preload라고만 설정하면 auto값을 기본으로 합니다.
      • poster 속성 : 브라우저 문제나 인터넷 연결 문제 등 비디오를 재생할 수 없을 경우, 비디오 화면 자리에 포스터 이미지를 대신 표시합니다.
        poster 속성은 다음과 같이 비디오 파일의 포스터 이미지를 지정합니다.

        ⟨ video src="media/Painting.mp4" controls poster="fireworks.jpg" ⟩⟨/video⟩

    • ⟨track⟩ 태그 : 비디오 화면에 자막 추가하기

      HTML5에서는 track 태그를 이용해 외부 자막 파일을 연결할 수 있습니다.
      일반적으로 자막은 .smi/.srt를 사용하지만, 웹에서는 srt 파일을 사용합니다. 또한 모든 웹에서 공식적으로 지원하는 자막 파일 형식은 WebVTT(.vtt) 파일입니다.
      기본형은 다음과 같습니다.

      ⟨ track kind="자막 종류" src="경로" srclang="언어" label="제목" default ⟩

      track 태그에서 사용할 수 있는 주요 속성은 다음과 같습니다.

      • kind : 자막의 종류를 지정합니다. 사용할 수 있는 값은 다음과 같으며 생략할 경우 subtitles로 간주합니다.
        • subtitles : 자막입니다. 소리를 켤 수 있지만 이해할 수 없는 경우, 다른 언어로 번역할 자막일 경우 사용하며 비디오 화면에 표시됩니다.
        • captions : 캡션입니다. 청각장애인용 자막이거나 소리를 들을 수 없거나 켤 수 없는 경우 사용합니다. 비디오 화면에 표시됩니다.
        • descriptions : 비디오 콘텐츠에 대한 설명입니다. 비디오 화면에는 표시되지 않습니다.
        • chapters : 비디오 탐색을 위한 장 제목입니다. 비디오 화면에는 표시되지 않습니다.
        • metadata : 비디오 콘텐츠 정보입니다. 비디오 화면에 표시되지 않습니다.
      • src : 자막 텍스트의 파일 경로를 지정합니다.
      • srclang : 사용한 언어를 지정합니다. kind 속성 값이 subtitle이라면 언어를 나타내는 약자[?]로 반드시 지정해야 합니다.
      • label : 자막이 여러 개일 경우, 자막을 식별할 수 있도록 제목을 달아 줍니다.
      • default : 자막 파일이 여러 개일 경우, 기본으로 사용할 자막을 지정해 줍니다.
