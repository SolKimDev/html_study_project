(기억이 잘 안 나는 녀석들도 포함합니다.)

챕터 2-2. 기본 태그

  • blockquote 태그는 블럭으로 구분된 인용 텍스트입니다. ⟨cite⟩ 속성을 이용해 인용 사이트 주소를 표시할 수 있으며, 들여쓰기를 통해 인용이 표현됩니다.
    이것이 ⟨q⟩ 태그와 다른 점은, ⟨q⟩ 태그의 경우 인라인 레벨 태그로 줄바꿈 없이, 따옴표를 이용해 인용임이 표시되지만 ⟨blockquote⟩ 태그의 경우 블록 레벨 태그이기 때문에 블록 구분을 통해 인용 내용이 구분된다는 점입니다. 

  • pre태그의 경우, 입력하는 그대로 화면에 표시해 주는 기능을 가지고 있습니다.
    
            js 코드 같은 것들은 그대로 표시할 수 있지만, HTML 태그같은 것은 적용이 되어 표현됩니다.
    
            ⟨code⟩나 ⟨samp⟩, ⟨kbd⟩ 등 코드와 관련된 내용을 표시할 때 이용하면 좋습니다.
    
            다만, 웹 접근성 부분에서 좋지 않기 때문에 사용에 있어서 주의가 요구됩니다.
    PRE 태그 사용시 주의 사항 :
    pre 태그는 그 특성상, 모바일에서 표시할 때에도 화면 크기를 무시하고 내용이 적힌 대로 표시하게 됩니다. 아마 추후에 화면 레이아웃을 지정할 때에도 이를 어길 가능성이 있다고 추측되므로, 사용 시 주의가 요구됩니다.


  • strong 태그는 ⟨b⟩ 태그와 마찬가지로 글자를 굵게 표시합니다.
    ⟨b⟩ 태그와의 차이점은 의미 중심적이라는 것으로, 덕분에 낭독기 접근성이 향상된다는 것입니다.

  • emem태그의 경우 ⟨strong⟩태그의 ⟨i⟩talic 버전이라고 이해할 수 있습니다.
    ⟨i⟩태그에 대한 의미론적 버전이라고 할 수 있겠네요.

  • q태그는 따옴표로 표시되는 인라인 레벨 태그입니다.

  • mark태그는 형광펜 효과를 내는 태그입니다. CSS의 background-color 속성을 사용해 배경 색상을 바꿀 수 있습니다.

  • span태그는 일부 영역에만 효과를 주기 위해 이용되는 범위 지정 인라인 태그입니다.
    인라인 대신 단락을 블록 단위로 적용하고 싶다면 ⟨div⟩ 태그를 이용합니다.
  • ruby 태그는 간단히 말해, 振ふり仮名がな(후리가나) 전용! 이라고 생각하면 될 듯 합니다.

  • abbr 태그는 title 속성을 함께 이용하여 약자를 표시하는데 이용합니다.
    abbr을 이용할 경우 텍스트 아래에 점선이 표시되지만, abbr이 아니더라도 해당 텍스트를 싸고 있는 태그에 타이틀을 적용할 경우, 마우스오버가 된 뒤 잠시 시간이 지나면 태그에 적용된 타이틀이 표시됩니다.
    다만 아직 모든 태그에 적용되는지는 테스트 해 보지 않았습니다.
    또 한 가지 주의할 점은, hierarchy에 따라 해당 오브젝트를 포함하는 상위의 태그에 타이틀이 적용되어 있으며, 해당 오브젝트에 타이틀이 적용되지 않은 경우, 상위 태그의 타이틀을 표시합니다.
    설명이 좀 별로긴하지만 C언어의 지역 변수 전역 변수 비슷한 규칙이라고 생각하면 될 것 같습니다.


  • cite 태그는 웹 문서나 포스트에서 참고 내용을 표시합니다. - https://www.w3schools.com/tags/tag_cite.asp

  • code 태그는 컴퓨터 인식을 위한 소스 코드를 적용합니다. 오작동 방지를 위해 ⟨pre⟩ 태그와 함께 이용할 것이 권장됩니다.

  • kbd 태그는 키보드 입력이나 음성 명령 같은 사용자 입력 내용을 표시합니다. 예를 들어, 현 페이지는 ' F5 ' 키를 누르면 새로고침됩니다.

  • small 태그는 부가 정보처럼 작게 표시해도 되는 텍스트를 표시하는 데 이용됩니다.

  • sub 태그와 sup 태그는 각각 아래 첨자sub와 위 첨자sup를 표현하는 데에 이용됩니다.

  • s 태그는 취소선을 긋는 데에 이용됩니다. 나무위키가 생각나면 당신은 이미 진 겁니다. - https://namu.wiki/w/취소선?from=취소선 드립#s-6

  • u 태그는 밑줄을 긋는 데에 이용됩니다. 링크인 척 색깔을 넣었지만 클릭해도 아무것도 연결되어 있지 않습니다. 눌러보려 했다면 이번에도 당신은 진 겁니다.

챕터 2-3. 목록을 만드는 태그
목록을 만드는 태그는 여러 가지가 있으며, 그 목록은 아래와 같습니다. A. B. C.와 같은 부분만 강조하려면 스타일시트를 이용하는 방법밖에 없을까요? 한 번 찾아 봅시다.
  1. ul & li
  2. ol & li
  3. dl & dt & dd
*tip : ⟨/li⟩태그는 생략 가능합니다.
  1. ul태그와 li태그는 순서가 없는 목록을 표시하고자 할 때에 이용됩니다.
    각 항목은 원이나 사각형 모양의 불릿(bullet)으로 표시되며, HTML5부터는 CSS의 "list-style-type" 속성을 이용해 불릿의 모양을 수정할 수 있습니다.
    • Unordered List가
    • 사용되는 모습
    • Chapter 02-2에도 li가 적용되었으나
    • 가장 상위의 불릿이기 때문에 불릿이 적용된 색상이나 모양이 다른 것으로 보인다.

  2. ol태그와 li태그는 순서가 있는 목록을 표시하고자 할 때에 이용되며, Chapter 02-3에 해당하는 해당 문서 역시, ol을 이용하고 있습니다.
    각 항목은 여러 속성이 함께 사용되어 목록의 숫자 표기 방법이나 시작하는 숫자 등을 바꿀 수 있습니다.
    1. type 속성 : 항목의 머리에 붙는 리스팅용 문자를 변경할 수 있습니다.
      • 숫자 : 기본값으로, 속성 값은 '1'입니다.
      • 영문 소문자 : 속성 값은 'a' 입니다.
      • 영문 대문자 : 속성 값은 'A' 입니다.
      • 로마자 소문자 : 속성 값은 'i' 입니다.
      • 로마자 대문자 : 속성 값은 'I' 입니다.
    2. start 속성 : 리스팅 숫자를 지정한 숫자부터 시작할 수 있게 하는 속성입니다.
    3. reversed 속성 : 항목을 역순으로 표시합니다.

  3. dl과 dt, dd 태그는 '제목'과'설명'이 한 쌍인 설명 목록(description list)를 만듭니다. 사용 예시는 다음과 같습니다.

    제주 올레
    올레 1코스
    코스 : 시흥 초등학교 옆 - 광치기 해변
    거리 : 14.6km(4~5시간)
    난이도 : 중
    올레 2코스
    코스 : 광치기 해변 - 온평 포구
    거리 : 14.5km(4~5시간)
    난이도 : 중
